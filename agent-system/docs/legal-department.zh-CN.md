# 企业法务部设计说明

数字办公室中的法务部不是律师事务所团队，也不是照搬人类公司层级的虚拟组织。它是一个企业内部的能力命名空间、权限边界、知识空间和工作流集合。

## 设计原则

- 部门是产品心智模型，运行时仍以 workflow 和能力节点为主。
- 法务负责人 Agent 只在 intake、分派、综合、升级和审批门禁处出现。
- 窄任务直达专业能力节点，避免每次都经过“层层汇报”。
- 所有法律输出都是内部审查草稿，不是正式法律意见或法律结论。
- 对外发送、签署、上线、解雇、诉讼、监管提交等动作必须经过人工专业审查。

## 部门结构

```text
秘书 Agent
  -> 企业法务部 legal
      -> 合同审查 legal-contracts
      -> 合规与产品法务 legal-compliance
      -> 法务部综合 legal
```

`legal` 是部门接口与最终综合节点；`legal-contracts` 负责合同、协议、尽调与交易支持；`legal-compliance` 负责隐私数据、产品上线、营销宣传、AI 治理、用工、知识产权、监管动态等合规事项。

## 常用工作流

- `legal_triage`：用户只说“找法务”或任务边界不清时使用。
- `legal_contract_review`：供应商合同、客户合同、SaaS、NDA、采购、服务协议、合同争议。
- `legal_privacy_data_review`：个人信息、数据处理、DPA、PIA、主体权利响应。
- `legal_product_compliance_review`：产品上线、营销宣传、AI 治理、监管变化、合规就绪。
- `legal_employment_ip_review`：劳动用工、解除、录用、知识产权、开源合规。
- `legal_dispute_triage`：律师函、争议、诉讼/仲裁前分流、外部律师协作。

## 外部技能库接入策略

`claude-for-legal-zh` 以 Apache 2.0 许可发布，登记为法务部候选安装源。它按商事合同、公司法务、隐私数据、产品法务、劳动用工、AI 治理、知识产权、监管合规、争议解决等领域组织，适合作为企业法务 workflow 的来源。

`Legal-Skills-Chinese` 提供中国法推理原子技能，但其 README/CONTRIBUTING 标注为 CC BY-NC-ND 4.0。当前只能登记为参考源，不能直接内置、改作、再分发或商用，除非后续完成授权确认。

## 交付标准

法务部最终交付必须包含：

1. 业务可读摘要
2. 来源与核验状态
3. 风险等级和影响
4. 建议动作和替代方案
5. 禁止动作或需暂停动作
6. 人工审查/审批门槛
