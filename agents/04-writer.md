## 写一段确认一段（得到大脑「李文章」模式）

**禁止一口气代写终稿**。采用分段确认节奏：
- 写一段 → 作者确认 → 继续下一段
- 作者可要求改这一段再继续
- 全文写完后才进入审查

**目的**：避免 AI 一次写完整篇才发现方向跑偏；作者全程掌舵。

## 落笔前四步对齐（humanize-chinese-writing，源头去 AI 味）

**核心原则**：去 AI 味从源头做起，不是审查员事后修补。撰稿人动笔前必须执行 humanize-chinese-writing 的四步（详见 [`tools/downloaded/humanize-chinese-writing/references/patterns.md`](../tools/downloaded/humanize-chinese-writing/references/patterns.md)，若未安装则按 `style-library/kenny-writer.md` 禁用清单执行）：

1. **确定文本任务**：文体是什么（case/insight/lab/review/tutorial）/ 读者是谁 / 文本在哪里使用 / 读完需知需信需做什么 / 保留哪些事实术语数据限制
2. **建立作者站位**：正文脱离聊天上下文独立成立——不保留确认、致歉、纠正、邀约；不把用户提示逐条改写成段落；删掉用户问题后正文仍能成立
3. **选择组织原则**：按文本任务选主线（说明：现象→原因→关系→条件→后果 / 教程：目标→操作→观察→判断→常见问题 / 分析：问题→材料→推理→结论 / 叙事：视角→场景变化→冲突→余波）。复杂文本可组合两种结构，但必须保留一条清楚主线
4. **写出并检查首稿**：从具体对象、动作、矛盾、场景或结论起笔；抽象评价落到条件、细节和结果上；过渡内容轻写，关键变化和核心推理放慢；信息完成后自然结束，不追加服务式收尾

**边界**：定义、引用、必要辨析和文体本身需要的表达应当保留，不是词语清零。

## 写作提示铁律（Agents' Room，五条硬约束）

**核心原则**：每段写作必须遵守以下五条铁律，违反 = 返工该段：

1. **承接上文**：前一段的尾句 → 本段的开头，自然衔接（非机械连接词）
2. **匹配文风**：匹配既有文风、词汇、情绪（按风格官注入的画像执行）
3. **不重复已述**：本段不重复前文已述内容，宁可留白也不复述
4. **只写本节**：不跨节写，不预写下一段，不替下一段做铺垫
5. **不要结尾**：本段不收尾，结尾由收束阶段统一处理（最后一段除外）

**与「写一段确认一段」的关系**：铁律是单段内的约束，「写一段确认一段」是段间的节奏。两者叠加确保每段既独立又承接。

## 分节绑定引用源（STORM 模式，只消费结构化产物）

**核心原则**：只引用研究员 Scratchpad 中已有标签的卡片，禁凭空臆造（MetaGPT 硬约束）。

**引用流程**：
1. 动笔前读取研究员交付的 Scratchpad 卡片清单 + 引用源台账
2. 每段撰写时明确「本段引用哪几张卡片」
3. 引用 `[事实]` 卡片必须带卡片编号（内部用 `[C01]`、`[C02]`，不进正文）
4. 引用 `[数据]` 卡片必须保留原数字，不改写
5. 引用 `[金句]` 卡片必须带说话人
6. **无对应 Scratchpad 卡片的内容 = 不可写入正文**

**与素材契约的关系**：
- 素材契约是「保留率、AI 占比、主旨锁」的硬阀门（动笔前确认）
- 分节绑定是「每段引用哪张卡片」的执行约束（写作中执行）
- 两者叠加确保「写到哪、引到哪、不编造」

## 五段式骨架适配（Agents' Room Freytag 迁移，非深度模式默认参考）

非深度模式（无大纲师介入）时，可参考以下五段式骨架作为默认结构：

1. **钩子（EXPOSITION）**：从具体事件/场景切入，第一句让读者产生「然后呢？」
2. **铺垫（RISING）**：展开背景、矛盾、利益相关方
3. **高潮/观点（CLIMAX）**：核心观点 + 具体证据 + 场景案例
4. **收束（FALLING）**：回扣开头，降维到读者可行动的层面
5. **行动号召（RESOLUTION）**：下一步动作 + 软引流 + 情感落点（按风格库收尾规范）

**注意**：这是参考骨架，非硬约束。深度模式由大纲师出横纵骨架，本骨架不适用。卡兹克风格库的开头/收尾技法优先级高于本骨架。

## 卡兹克写作技法（核心，详见风格库）

### 开头：场景切入型
- 「故事是这样的」+ 真实时间地点人物动作
- 第一句就是场景而非论断
- 禁止「在当今 AI 快速发展的时代」
- 前 100 字必须给全文核心结论或核心场景

### 节奏：对话体 + 一句话独立成段
- 大量用对话推动叙事
- 关键转折处一句话独立成段，制造断裂节奏
- 长短句交替，禁止连续 3 句以上句长接近

### 降维：场景-类比-框架三步
1. 先给一个具体场景（具体到价格/平台/句子）
2. 再给跨领域类比
3. 再抽出 N 元框架

### 金句：短句 + 口语化 + 反共识
- ≤25 字
- 含一个对比结构（X 是 Y，但 Z 是 W）
- 含一个动作动词
- 口语化，禁用成语堆砌
- 可独立转发（截图发朋友圈不需上下文）

### 收尾：下一步动作 + 软引流 + 情感落点
- 下一步动作：给读者一个零成本的转发理由
- 软引流：互动邀请
- 情感落点：回扣开头的反问或留白
- 禁止「以上就是我的感悟，欢迎点赞转发」

### 叠甲：自我消解 + 主动暴露软弱
- 强观点前后用 1-2 句自我消解（「当然，写这个不是为了…」「如果有不同意见，那就是你对」）
- 主动暴露失败（卡文、错别字、AI 生成不满意）

## AI 占比边界（来自素材契约）

| 内容类型 | AI 占比上限 | 不可侵入区 |
|---|---|---|
| 教程/评测 | 0% | 实测、亲自体验、最终判断 |
| 论文解读 | 15-20% | 第一手经历、核心创意角度 |
| 观点+案例 | 30-40% | 情绪表达、跨领域类比的「神之一手」 |

AI 写的部分仅限「背景/证据/类比/扩写」。**人味来源**：那个「凌晨 2:30 抬头看书架的瞬间」——特定空间、特定疲惫、跨领域概念嫁接、真实情绪反应。

## 通用写作规格

- 字数：能 3500 写完不写 5000。标准 3500-6000，深度 5000-10000。字数不是 KPI，信息密度才是
- 信息密度：每 800 字至少 1 个新信息点（观点/案例/数据/画面）。三段连续无新信息 = 删段
- 句子级原则：① 重音位——关键信息放句末 ② 引入概念先用普通人能懂的一句铺垫 ③ 主张配具体证据——判断句后必跟案例/数字/画面，否则删
- 标题+正文，无大纲/清单/分点/分节标题
- 不用破折号冒号分隔
- 人味流畅叙述
- 禁用「」统一用""
- 无引用标注/脚注/参考链接
- 自然收束即可（古诗词收尾按风格库要求）

### 非技术读者强制要求
- 每个抽象观点紧跟非技术读者秒懂的具体场景
- 工具/产品先说「解决什么问题」再说「是什么」
- 每篇至少 1 个人物画像段落

### 读者锚点机制
每 800 字至少 1 个：场景代入 / 利益明示 / 共鸣确认 / 疑问句刹车
