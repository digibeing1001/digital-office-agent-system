## Step 3.5 横纵骨架（仅深度模式）

**触发**：用户在 Step 1/2 说过「写深一点」「横纵分析」「深度文」「5000-10000 字」等。

按横纵方法论，用三段式骨架组织已确认的四维素材：

```
🔭 横纵骨架
【纵向脉络】
- 起源：...
- 关键节点 1（YYYY）：...（核心特征 + 核心矛盾）
- 关键节点 2（YYYY）：...
- 当下阶段：...（3-4 个阶段，每个有核心特征和核心矛盾）
【横向格局】
- 竞品 A：核心差异 / 用户口碑 / 生态位
- 竞品 B：...
【横纵交汇】
- 综合判断 1：历史如何塑造当下
- 综合判断 2：优势的历史根源
- 综合判断 3：劣势的历史根源
- 未来三剧本：最可能 / 最危险 / 最乐观
```

⛔ 骨架未经作者拍板前不得进入 Step 4。骨架是「素材组织方式」，不是「写作风格」——最终成稿仍由撰稿人接管，咨询报告腔被人味消解。

## 分层大纲生成（STORM 模式，把横纵骨架转化为分层 JSON）

骨架拍板后，大纲师把横纵骨架转化为**分层结构化 JSON**，入 Scratchpad 共享区，供撰稿人按标签引用：

```json
{
  "title": "{标题}",
  "type": "{case/insight/lab/review/tutorial}",
  "structure": "横纵骨架",
  "sections": [
    {
      "section_id": "S1",
      "label": "[钩子]",
      "heading": "{章节标题}",
      "word_target": {字数预期},
      "scratchpad_refs": ["C01", "C03"],
      "key_points": ["{要点1}", "{要点2}"]
    },
    {
      "section_id": "S2",
      "label": "[纵向-起源]",
      "heading": "{章节标题}",
      "word_target": {字数预期},
      "scratchpad_refs": ["C02", "C05"],
      "key_points": ["{要点1}"]
    }
  ]
}
```

**字段说明**：
- `section_id`：章节编号，撰稿人按编号顺序写
- `label`：标签分区（与研究员 Scratchpad 标签对齐：`[事实]`/`[数据]`/`[金句]`/`[案例]`/`[类比]`/`[观点]`）
- `scratchpad_refs`：本节引用哪些研究员卡片编号
- `key_points`：本节必须覆盖的要点

**硬约束**：
- 每节必须标注 `scratchpad_refs`，无引用 = 该节不可写
- 撰稿人按 `section_id` 顺序写，不跨节
- 大纲师只产结构化 JSON，不写正文片段（Agents' Room 模式）

## 五段式骨架适配（Agents' Room Freytag 迁移，非深度模式可选参考）

非深度模式（无横纵骨架）时，大纲师可输出五段式骨架作为默认结构参考：

| 段 | 名称 | 作用 | 字数占比 |
|---|---|---|---|
| 1 | 钩子（EXPOSITION） | 从具体事件/场景切入，第一句让读者产生「然后呢？」 | 10-15% |
| 2 | 铺垫（RISING） | 展开背景、矛盾、利益相关方 | 20-25% |
| 3 | 高潮/观点（CLIMAX） | 核心观点 + 具体证据 + 场景案例 | 35-40% |
| 4 | 收束（FALLING） | 回扣开头，降维到读者可行动的层面 | 15-20% |
| 5 | 行动号召（RESOLUTION） | 下一步动作 + 软引流 + 情感落点 | 5-10% |

**注意**：这是参考骨架，非硬约束。卡兹克风格库的开头/收尾技法优先级高于本骨架。

## 结构路由（按文章类型）

| 类型 | 推荐结构 | 说明 |
|---|---|---|
| case | STAR（情境-任务-行动-结果） | 产品体验型，落地说服力 |
| insight | SCQA + 金字塔 | 现象解读型，结论先行 |
| lab | 钩子-探索-发现-反思 | 调查实验型，过程感 |
| review | 钩子-痛点-方案-证据 | 决策型，工具选择 |
| tutorial | PREP 循环（点-理-例-点） | 教学型，可复制步骤 |

- 骨架是段落分配地图，写作风格由撰稿人决定
- 历史对照优先调用历史对照参考库
