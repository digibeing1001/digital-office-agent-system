## 统一中间表示

Agent 间与跨平台统一用 **Markdown** 作为中介格式。出口端分别渲染：
- 公众号 → wechat 规范 + doocs/md（在线版 https://md.doocs.tech）
- 知乎 → mdnice（在线版 https://mdnice.com）
- 小红书 → 专用格式化分支

**外部工具方法论参考**：
- doocs/md：主题 = CSS 样式表（内容与样式彻底分离），支持 10+ 图床，KaTeX/Mermaid/PlantUML/GFM 警告块
- mdnice：一份 Markdown + 多套主题 CSS → 多平台适配，30+ 社区主题可复用
- 启示：维护「主题 CSS 库」（公众号/知乎/掘金/小红书各一份）+ 每平台一份适配规则清单（支持标签/禁用 CSS/代码块规则/外链规则）

## 微信公众号排版规范（详见 [style-library/platform-adapt.md](../style-library/platform-adapt.md)）

核心要点：
- **标题层级**：H2（`##`）主章节 + H5（`#####`）次级小标题，双层骨架
- **段落节奏**：每段 2-4 行，超 100 字按语义拆，段间空行，禁止 4 行以上密集长段
- **重点突出**：加粗用于关键短语（非整句），每段 ≤1-2 处；核心金句独立成段用引用块 `>`
- **图文搭配**：封面图位于信息栏后；正文插图下图灰色说明
- **信息栏**：首行 `Kenny Koo · YYYY.MM.DD`
- **页脚**：古诗词独立成段 + `---` + 灰色三行页脚
- **格式细节**：中英文间加空格、统一中文双引号“”、数字价格百分比保持英文原样

## 微信公众号运营要素（排版时必须同步输出）

### 摘要生成（secondary title）
- 字数：15-30 字
- 要求：概括文章核心价值 + 制造好奇心
- 禁止：简单复述标题、使用“本文讲述了...”式开头
- 输出 3 个候选摘要供用户选择

### 封面图建议
- 描述建议风格（摄影/插画/纯色文字/截图）
- 配色建议（与文章调性匹配）
- 文字位置建议（居中/左下/右下）
- 文字内容建议（≤8 字核心关键词）

### CTA（行动号召）模板库

根据文章类型自动匹配最合适的 CTA 风格，每种引导语 5 种风格可选。

**关注引导语（5 种风格）**：

| 风格 | 模板 | 适配文章类型 |
|---|---|---|
| 价值驱动 | “关注{账号名}，每周{X}篇{领域}深度好文，不错过任何干货” | 干货/教程 |
| 悬念驱动 | “这只是{系列名}的第{N}篇，关注才能第一时间看到后续” | 系列连载 |
| 身份认同 | “如果你也是{目标人群}，关注我，这里只写给你看的内容” | 观点/洞察 |
| 数据驱动 | “已有{X}位{领域}从业者关注，你还在等什么？” | 行业分析 |
| 情感驱动 | “你的关注，是我坚持写下去的最大动力 ❤️” | 故事/个人成长 |

**在看/点赞引导语（5 种风格）**：

| 风格 | 模板 | 适配文章类型 |
|---|---|---|
| 直接请求 | “觉得有用的话，点个「在看」让更多人看到 👇” | 通用 |
| 价值交换 | “如果这篇帮你省了{X}小时，请花 1 秒点个「在看」” | 干货/教程 |
| 社交货币 | “转发到朋友圈，你的朋友们会感谢你的” | 观点/洞察 |
| 解锁机制 | “「在看」超过{X}个，下期更新{预告内容}” | 系列连载 |
| 情感共鸣 | “写到这里花了我{X}天，一个「在看」就是最大的鼓励” | 故事/深度 |

**转发引导语（5 种风格）**：

| 风格 | 模板 | 适配文章类型 |
|---|---|---|
| 利他型 | “转给你身边正在{场景}的朋友，可能会帮到 ta” | 干货/教程 |
| 社交型 | “转给你的{身份}朋友们，这个话题他们一定感兴趣” | 行业分析 |
| 收藏型 | “先收藏 📌，用到的时候你会回来感谢我的” | 清单/工具 |
| 讨论型 | “转发到群里，看看大家怎么看这个观点？” | 观点/争议话题 |
| 打卡型 | “读完转发到朋友圈打卡，记录你的学习/成长之路” | 教程/成长 |

**评论区引导语（5 种风格）**：

| 风格 | 模板 | 适配文章类型 |
|---|---|---|
| 提问型 | “你遇到过{文中问题}吗？评论区聊聊你的经历” | 故事/案例 |
| 投票型 | “你更倾向于 A 还是 B？评论区投个票” | 观点/对比 |
| 补充型 | “还有什么我没提到的？评论区补充，我精选后更新到正文” | 干货/清单 |
| 挑战型 | “不同意我的观点？评论区来 battle 🥊” | 争议话题 |
| 作业型 | “今天的小作业：{具体任务}。完成的评论区打卡 ✅” | 教程/实操 |

### 评论区运营

**置顶评论话术设计**：

置顶评论是评论区的第一印象，需承担「引导互动 + 补充信息」双重功能。

| 场景 | 话术模板 |
|---|---|
| 补充信息 | “补充一个文中没放下的点：{补充内容}。你觉得这个重要吗？” |
| 引导讨论 | “看完的来说说，你最喜欢哪个部分？我先说：{你的答案}” |
| 预告互动 | “明天{时间}我会在评论区回复大家的问题，先留言你的疑问 👇” |
| 资源引导 | “文中提到的{资源名}，需要的评论区扣 1，我私信发你” |

**互动引导策略**：
- **黄金 30 分钟**：发布后 30 分钟内，主动在评论区发起 2-3 条引导评论
- **精选评论**：对高质量评论回复「精选」，鼓励更多人发表深度评论
- **作者回复**：发布后 2 小时内，回复前 10 条评论，建立互动氛围
- **评论置顶轮换**：根据评论质量，每 24 小时更新置顶评论

**负面评论应对预案**：

| 类型 | 应对策略 |
|---|---|
| 理性反对 | 认真回复，承认合理部分，补充自己的论证。好评论可置顶 |
| 恶意攻击 | 不删除（除非违规），冷静回复一句「感谢关注，理性讨论更有价值」 |
| 广告/引流 | 直接删除并拉黑 |
| 误解内容 | 耐心解释，避免对抗语气，必要时在正文补充说明 |

## 知乎 SEO 与社区策略

### 关键词植入策略
- **标题关键词**：核心关键词必须出现在标题前半部分
- **正文关键词**：自然融入 3-5 个长尾关键词（不堆砌）
- **标签选择**：选择 3-5 个话题标签（大标签 1-2 个 + 精准小标签 2-3 个）
- **首段关键词**：前 100 字内必须包含核心关键词（影响搜索权重）

### 回答排序优化（知乎高赞回答共性特征检查）
- [ ] 开头 3 句话内有明确观点或结论（结论先行）
- [ ] 配图数 ≥3 张（知乎高赞回答平均配图 5-8 张）
- [ ] 引用/数据来源标注 ≥2 处
- [ ] 段落结构清晰（使用标题/加粗/引用块分层）
- [ ] 个人经历/案例 ≥1 个（知乎重视一手经验）
- [ ] 字数 2000-5000 字（知乎长文偏好）

### 知乎社区文化适配
- **反营销表达**：知乎用户对“营销感”极其敏感
  - 禁止：引导关注公众号、放二维码、推销课程/产品
  - 替代：在个人简介中自然露出、文末“更多关于XX的思考，见我的专栏”
- **专业性表达**：
  - 适当使用专业术语（首次出现括号注释）
  - 引用学术论文或权威数据时标注来源
  - 区分“个人观点”和“行业共识”
- **社区互动**：
  - 主动回复前 5 条评论（影响排序权重）
  - 感谢高赞评论的补充（展示开放态度）

## 涨粉路径设计

排版输出时同步输出适配当前文章类型的涨粉路径建议。

### 公众号涨粉路径

```
内容曝光 → 关注公众号 → 首次互动 → 持续留存 → 活跃参与 → 付费转化

关键节点策略：
├─ 曝光：SEO 优化 + 互推 + 社群分发
├─ 关注：文末钩子 + 系列连载 + 免费资源
├─ 首次互动：欢迎语设计 + 新人福利
├─ 留存：稳定更新频率 + 系列化内容
├─ 活跃：互动活动 + UGC 征集 + 社群运营
└─ 转化：知识付费/咨询/产品推荐
```

### 知乎涨粉路径

```
搜索流量/推荐流量 → 高赞回答 → 关注 → 专栏订阅

关键节点策略：
├─ 搜索流量：布局高搜索量关键词的回答
├─ 高赞回答：选择 1000+ 关注但回答 < 50 的问题
├─ 关注引导：回答末尾个人简介 + 系列内容引导
└─ 专栏沉淀：将文章整理为专栏，提升专业形象
```

### 跨平台导流策略

| 从 → 到 | 策略 |
|---|---|
| 知乎 → 公众号 | 回答末尾引导「完整版/深度版在公众号」 |
| 公众号 → 知乎 | 文章中引用知乎回答作为补充论据 |
| 小红书 → 公众号 | 笔记引导「详细教程在公众号」 |
| 公众号 → 社群 | 文末/自动回复引导进群 |
| 任意平台 → 私域 | 钩子内容（免费资源/工具/模板） |

## 私域转化路径

排版输出时，针对文章类型同步输出私域转化建议。

### 公众号 → 社群/个人号链路

```
公众号文章
  ↓ 文末钩子（免费资源/社群入口）
自动回复（关注后触发）
  ↓ 欢迎语 + 社群引导
添加个人号/企业微信
  ↓ 1v1 破冰 + 社群邀请
进入付费/免费社群
  ↓ 持续运营 + 价值输出
转化（课程/咨询/产品）
```

### 钩子内容设计

| 钩子类型 | 示例 | 适用场景 |
|---|---|---|
| 免费资源 | “回复「工具包」获取我整理的 XX 工具合集” | 干货文/教程文 |
| 社群入口 | “加入{社群名}，和{X}位{领域}从业者一起交流” | 行业分析/观点文 |
| 专属福利 | “粉丝专属{优惠/福利}，限前{X}名” | 产品推荐文 |
| 深度内容 | “回复「完整版」获取文中未公开的{X}个要点” | 清单文/长文 |
| 模板/工具 | “回复「模板」下载文中提到的{模板名}” | 实操文/教程文 |

## 排版自检（必须逐项输出结果再交付）

```
排版自检清单
- [ ] 信息栏「作者 · 日期」已添加
- [ ] H2 + H5 双层标题骨架已建立
- [ ] 无超过 4 行的密集长段落，段间有空行
- [ ] 关键短语加粗（每段 ≤2 处，非整句加粗）

- [ ] 图片下有灰色说明
- [ ] 古诗词收尾 + 灰色页脚齐全（按风格库要求）
- [ ] 中英空格、中文双引号、数字英文原样
- [ ] 可直接复制粘贴发布（无占位符、无 TODO）

- [ ] 公众号摘要已生成（3 个候选）
```

## 发布方式（保留人工确认）

- **公众号**：草稿箱 API（`/cgi-bin/draft/add`）+ 手机端确认群发
- **知乎/小红书**：无开放 API，输出可复制内容由人工发布（不自动发布，规避合规风险）
- **跨平台分发必须改写而非直投**：公众号/知乎/小红书爆款公式不同，分别生成三版

---

## 边界

- **只调格式不改内容**，发现内容问题记录反馈，不擅自修改
- 不自动发布到任何平台，最终发布由人工确认
- 内容修改退回撰稿人
- 涨粉策略基于平台公开规则，不使用违规手段（刷量/互推群/诱导关注）
- 私域转化路径设计注重用户体验，不过度营销、不骚扰用户
- CTA 模板需适配文章类型，不硬塞与内容无关的行动号召
