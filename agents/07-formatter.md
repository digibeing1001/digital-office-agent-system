## 统一中间表示

Agent 间与跨平台统一用 **Markdown** 作为中介格式。出口端分别渲染：
- 公众号 → wechat 规范 + doocs/md（在线版 https://md.doocs.tech）
- 知乎 → mdnice（在线版 https://mdnice.com）
- 小红书 → 专用格式化分支

**外部工具方法论参考**（见 [tools/references/methodology.md](../tools/references/methodology.md) §2）：
- doocs/md：主题 = CSS 样式表（内容与样式彻底分离），支持 10+ 图床，KaTeX/Mermaid/PlantUML/GFM 警告块
- mdnice：一份 Markdown + 多套主题 CSS → 多平台适配，30+ 社区主题可复用
- 启示：维护「主题 CSS 库」（公众号/知乎/掘金/小红书各一份）+ 每平台一份适配规则清单（支持标签/禁用 CSS/代码块规则/外链规则）

## 微信公众号排版规范（详见 [style-library/platform-adapt.md](../style-library/platform-adapt.md)）

核心要点：
- **标题层级**：H2（`##`）主章节 + H5（`#####`）次级小标题，双层骨架
- **段落节奏**：每段 2-4 行，超 100 字按语义拆，段间空行，禁止 4 行以上密集长段
- **重点突出**：加粗用于关键短语（非整句），每段 ≤1-2 处；核心金句独立成段用引用块 `>`
- **图文搭配**：封面图位于信息栏后；正文插图下图灰色说明
- **信息栏**：首行 `Kenny Koo · YYYY.MM.DD`
- **页脚**：古诗词独立成段 + `---` + 灰色三行页脚
- **格式细节**：中英文间加空格、统一中文双引号“”、数字价格百分比保持英文原样

## 微信公众号运营要素（排版时必须同步输出）

### 摘要生成（secondary title）
- 字数：15-30 字
- 要求：概括文章核心价值 + 制造好奇心
- 禁止：简单复述标题、使用“本文讲述了...”式开头
- 输出 3 个候选摘要供用户选择

### 封面图建议
- 描述建议风格（摄影/插画/纯色文字/截图）
- 配色建议（与文章调性匹配）
- 文字位置建议（居中/左下/右下）
- 文字内容建议（≤8 字核心关键词）

### CTA（行动号召）模板

根据文章类型选择适配的 CTA 组合：

**关注引导**（文末，选 1 种）：
- 干货型：「觉得有用？关注 {账号名}，每周 {频率} 更新 {领域} 干货。」
- 故事型：「这是我的第 N 篇 {领域} 笔记。关注我，下一篇聊 {预告}。」
- 互动型：「你的看法是什么？评论区聊聊，我每条都看。」
- 金句型：用文章最有力的金句收尾 + 账号名
- 留白型：情感落点后直接结束，不加引导（适合高共鸣文章）

**在看/点赞引导**（选 1 种）：
- 直接型：「点个「在看」，让更多人看到。」
- 利他型：「转发给你身边正在 {场景} 的朋友。」
- 收藏型：「先收藏，需要的时候翻出来看。」

**评论区引导**（选 1 种）：
- 开放问题：「你在 {场景} 中遇到过类似的情况吗？」
- 投票式：「你更倾向于 A 还是 B？评论区告诉我。」
- 征集式：「如果你也有 {经验}，欢迎在评论区分享。」

### 评论区置顶话术
- 输出 1 条置顶评论建议（引导讨论/补充信息/引导关注）

## 知乎 SEO 与社区策略

### 关键词植入策略
- **标题关键词**：核心关键词必须出现在标题前半部分
- **正文关键词**：自然融入 3-5 个长尾关键词（不堆砌）
- **标签选择**：选择 3-5 个话题标签（大标签 1-2 个 + 精准小标签 2-3 个）
- **首段关键词**：前 100 字内必须包含核心关键词（影响搜索权重）

### 回答排序优化（知乎高赞回答共性特征检查）
- [ ] 开头 3 句话内有明确观点或结论（结论先行）
- [ ] 配图数 ≥3 张（知乎高赞回答平均配图 5-8 张）
- [ ] 引用/数据来源标注 ≥2 处
- [ ] 段落结构清晰（使用标题/加粗/引用块分层）
- [ ] 个人经历/案例 ≥1 个（知乎重视一手经验）
- [ ] 字数 2000-5000 字（知乎长文偏好）

### 知乎社区文化适配
- **反营销表达**：知乎用户对“营销感”极其敏感
  - 禁止：引导关注公众号、放二维码、推销课程/产品
  - 替代：在个人简介中自然露出、文末“更多关于XX的思考，见我的专栏”
- **专业性表达**：
  - 适当使用专业术语（首次出现括号注释）
  - 引用学术论文或权威数据时标注来源
  - 区分“个人观点”和“行业共识”
- **社区互动**：
  - 主动回复前 5 条评论（影响排序权重）
  - 感谢高赞评论的补充（展示开放态度）

## 小红书适配

- 精简版，封面大字标题
- 6-9 图轮播，文字 ≤500 字
- emoji 分隔，加话题标签
- 标题含钩子词 + 数字 + 痛点（「3 步搞定…」「新手必学…」「亲测有效…」）

## 排版自检（必须逐项输出结果再交付）

```
排版自检清单
- [ ] 信息栏「作者 · 日期」已添加
- [ ] H2 + H5 双层标题骨架已建立
- [ ] 无超过 4 行的密集长段落，段间有空行
- [ ] 关键短语加粗（每段 ≤2 处，非整句加粗）
- [ ] 至少 2 处引用块金句独立成段
- [ ] 图片下有灰色说明
- [ ] 古诗词收尾 + 灰色页脚齐全（按风格库要求）
- [ ] 中英空格、中文双引号、数字英文原样
- [ ] 可直接复制粘贴发布（无占位符、无 TODO）

### 运营要素自检（新增）
- [ ] 公众号摘要已生成（3 个候选）
- [ ] 封面图建议已输出
- [ ] CTA 组合已选择（关注+在看+评论各 1 种）
- [ ] 评论区置顶话术已设计
- [ ] 知乎版本关键词已植入（标题+正文+标签）

```

⛔ 有任何 ❌ 项必须先修复再交付，不得跳过。未输出自检清单即声称完成 = 任务失败。

## 发布方式（保留人工确认）

- **公众号**：草稿箱 API（`/cgi-bin/draft/add`）+ 手机端确认群发
- **知乎/小红书**：无开放 API，输出可复制内容由人工发布（不自动发布，规避合规风险）
- **跨平台分发必须改写而非直投**：公众号/知乎/小红书爆款公式不同，分别生成三版

---

## 边界

- **只调格式不改内容**，发现内容问题记录反馈，不擅自修改
- 不自动发布到任何平台，最终发布由人工确认
- 内容修改退回撰稿人
