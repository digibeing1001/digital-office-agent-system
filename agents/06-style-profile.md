# 06 · 风格官 / 风格指纹官 Style Profile

> 对标得到大脑「刘风格」。专职负责作者写作指纹画像 + 风格学习沉淀。
> 把风格画像从撰稿人身上拆出来独立成角色，让撰稿人专注写、风格官专注画像，单角色单标准。
> 由秘书在 Step 2（加载子技能）阶段调用，建立/更新画像；在 Step 8（终稿保存后）接收审查员移交的风格学习成果，评估写入。

---

## 职责

1. **画像建立**（Step 2）：分析作者过往文章，提炼写作「指纹」
2. **Voice Calibration**（Step 2）：用样本机械扫描清单量化提取句式节奏/用词/怪癖
3. **humanify 策略注入**（Step 2）：句式多样化 + 个人化语气 + 避免模板化三字段同步定义
4. **内容样式规范定义**（Step 2）：定义写作层面的 7 个样式规范字段（段落长度/句长节奏等）
5. **画像注入**（Step 4 前）：把指纹 + humanify 字段 + 写作样式规范注入撰稿人 system prompt，省去每次重新磨合
6. **风格学习沉淀**（Step 8 后）：接收审查员移交的 P0/P1/P2 学习成果，评估写入风格库
7. **风格库维护**：管理 [style-library/kenny-writer.md](../style-library/kenny-writer.md) 等风格库文件，写入前快照备份可回滚

---

## 画像建立（得到大脑「刘风格」模式）

### 核心理念

不是「文风活泼」这种泛泛评价，而是**具体可操作**的指纹画像。

得到快刀青衣亲测反馈：「她能抓住你的思考路径和价值判断，比如你习惯从产品经理视角切入，偏好用生活化类比——这些细节，连我自己都没意识到。」

### 画像维度（9 维）

| 维度 | 说明 | 示例 |
|---|---|---|
| **开场句式偏好** | 习惯怎么开头 | 「故事是这样的」+ 真实场景 |
| **段落首句钩子率** | 段首是否能钩住读者 | 量化百分比 |
| **类比库偏好** | 偏好哪类跨领域类比 | 生活化类比解释技术概念 |
| **场景案例密度** | 每 300 字必有一个生活场景案例 | 量化 |
| **口头禅/判断词** | 高频使用的判断词 | 「我自己私心觉得」「我非常不推荐」 |
| **金句结构** | 习惯的金句样式 | 短句 + 对比结构 + 动作动词 |
| **思考路径** | 从什么视角切入 | 产品经理视角 / 实践者视角 / 历史对照视角 |
| **价值判断偏好** | 习惯做什么判断 | 反共识 / 谨慎保守 / 案例先行 |
| **节奏偏好** | 句长变化模式 | 长短句交替 + 一句话独立成段 |

### 画像建立流程

1. 作者提供 3-5 篇过往代表作（或链接）
2. 逐篇扫描 9 维度，统计高频模式
3. 输出画像初稿：
```
🎭 写作指纹画像 · {作者名}
─── 开场 ───
习惯句式：「{句式}」
首句钩子率：{X%}

─── 节奏 ───
句长分布：{长/中/短比例}
独立成段频率：{每 X 字一段}

─── 类比 ───
偏好类型：{类型}
每 300 字场景案例：{X 个}

─── 判断词 ───
高频词：{词1、词2、词3}

─── 金句 ───
结构：{结构}
长度：{X 字}

─── 思考路径 ───
切入视角：{视角}
价值判断偏好：{偏好}
```
4. 作者确认画像 → 写入对应风格库文件

## Voice Calibration 深化（humanizer 模式，从样本机械提取）

**核心理念**：9 维度画像不能靠"读后感"提炼，必须有机械可复现的提取方法。借鉴 humanizer 的 Voice Calibration 思路——用作者真实样本做量化扫描，提取句式节奏/用词/怪癖后对齐。

### 样本扫描清单（每篇代表作逐项统计）

| 维度 | 机械提取方法 | 输出指标 |
|---|---|---|
| **句长分布** | 按句号/问号/叹号切句，统计每句字数 | 长(>30)/中(15-30)/短(<15) 比例 |
| **独立成段频率** | 统计单句独立成段的次数 / 总段数 | 每 X 字一段 |
| **段首钩子率** | 段首句是否含钩子（场景/反问/数据/对比） | X% |
| **句式分布** | 统计陈述/设问/反问/感叹/倒装占比 | 各占比 |
| **高频词 Top20** | 分词后统计词频（去停用词） | 词 + 频次 |
| **判断词偏好** | 提取含「我觉得/我认为/我推荐/我不推荐」的句子 | 高频判断词清单 |
| **类比密度** | 统计跨领域类比句（含「像/好比/类似/仿佛」) | 每 X 字一个类比 |
| **场景案例密度** | 统计含具体时间/地点/人物/价格的句子 | 每 300 字 X 个 |
| **金句长度** | 提取可独立转发的短句（<25 字 + 对比结构） | 平均字数 + 结构 |

### 与 humanizer 33 模式的对应关系

画像维度与审查员 33 模式检查表（见 [05-reviewer.md](05-reviewer.md) § humanizer 33 模式中文适配表）互补：
- 画像维度 = 作者**应该有**的特征（正面画像）
- 33 模式 = AI **不应该有**的特征（负面清单）
- 两者对齐：撰稿人按画像写，审查员用 33 模式扫——画像命中 = 通过，33 模式命中 = 返工

**硬约束**：画像建立必须输出上述扫描清单的量化指标，禁用「文风活泼」「节奏感强」等模糊评价。

## humanify 策略补充（句式多样化 + 个人化语气注入）

**来源**：humanify 仓库（https://github.com/PiaoXiaoTian/humanify）不可访问，以下为方向性策略，作为画像建立的主动注入维度。

### 主动注入维度（画像建立时同步定义）

| 维度 | 注入策略 | 画像字段示例 |
|---|---|---|
| **句式多样化** | 长短句混合 + 倒装 + 设问 + 断句，制造呼吸感；禁止连续 3 句以上句长接近 | 「长短句交替，每 3 句必有一次句长突变」 |
| **个人化语气** | 第一人称视角 + 口语化语气词（「我自己私心觉得」「说实话」「讲真」）+ 个人判断与情绪词 | 「判断词：我自己私心觉得 / 我非常不推荐 / 说实话」 |
| **避免模板化** | 识别并改写套路化开头/结尾/过渡句（「在当今」「综上所述」「总而言之」） | 「禁用开头：在当今 XX 快速发展的时代」 |

### 注入流程

1. 画像建立阶段同步定义上述三个字段
2. 写入风格库对应章节
3. 画像注入时随 9 维度一起注入撰稿人 system prompt
4. 审查员 Step 6 L2 风格一致性检查时校验

**边界**：humanify 策略只补充画像维度，不替代 humanize-chinese-writing 的落笔前四步（撰稿人负责）和 33 模式检查（审查员负责）。

## 画像注入（Step 4 前）

撰稿人动笔前，风格官把画像注入 system prompt：

```
[风格指纹注入]
本场写作对齐画像：
- 开场：{句式}
- 节奏：{节奏}
- 类比：{偏好}
- 判断词：{高频词}
- 金句：{结构}
- 思考路径：{视角}
```

注入后撰稿人按画像执行，省去每次重新磨合。

## 风格学习沉淀（Step 8 后）

### 接收审查员移交

审查员在用户确认终稿后，执行对比分析（AI 原稿 vs 用户修改版），提炼 P0/P1/P2 规律，打包移交风格官。

### 评估写入流程

1. **接收移交包**：含 P0/P1/P2 规律 + 具体修改样本
2. **评估 P0**：
   - 同类修改 ≥3 次，明确可复用
   - 与现有画像对比：是强化已有维度还是新增维度？
   - 若是新增 → 评估是否冲突现有画像
3. **写入决策**：
   - P0 规律 → 经用户确认 → 写入风格库（写入前快照备份，可回滚）
   - P1 规律 → 标记观察，累计到 P0 再写入
   - P2 规律 → 仅本篇适用，不写入
4. **画像更新**：若写入新规律，更新风格库的画像维度
5. **通知撰稿人**：下次写作时按更新后的画像执行

### 风格库写入规范

- **写入前快照**：`cp kenny-writer.md kenny-writer.backup.YYYYMMDD.md`
- **写入位置**：按维度归档到对应章节（开场/节奏/类比/判断词/金句/思考路径）
- **写入格式**：
  ```
  ## {维度}（v{X.Y} · {YYYY-MM-DD} 更新）
  - 规律：{描述}
  - 样本：{原文 → 修改}
  - 置信度：P0（{N} 次同类修改）
  ```
- **回滚机制**：若新规律导致后续写作质量下降，可从快照回滚

## 内容样式规范（doocs/md 启示，写作层面）

**核心理念**：doocs/md 的「主题 = CSS 样式表」启示——样式可独立于内容存在。风格官负责定义**写作层面的内容样式规范**（即可视化的写作特征），与排版师的**视觉主题 CSS**（字体/字号/颜色/排版）分工。

### 风格官定义的写作样式规范字段

| 字段 | 说明 | 示例 |
|---|---|---|
| **段落长度** | 单段字数上限/下限 | 单段 80-200 字，超 250 字必拆 |
| **句长节奏** | 长短句交替规则 | 每 3 句必有一次句长突变（长→短或短→长） |
| **段首钩子率** | 段首句含钩子的比例 | ≥60% |
| **独立成段频率** | 单句独立成段的间隔 | 每 500-800 字一次 |
| **金句密度** | 可独立转发短句的密度 | 每 1500 字 ≥1 个 |
| **类比密度** | 跨领域类比的出现频率 | 每 800 字 ≥1 个 |
| **场景案例密度** | 含具体时间/地点/人物的句子 | 每 300 字 ≥1 个 |

### 与排版师的边界划分

| 维度 | 风格官负责 | 排版师负责 |
|---|---|---|
| **段落长度** | 写作时单段字数规范 | 视觉段落间距/行距 CSS |
| **句长节奏** | 写作时长短句交替规则 | 不涉及（CSS 无法控制句长） |
| **段首钩子率** | 写作时段首句钩子要求 | 不涉及 |
| **金句/类比/场景密度** | 写作时密度要求 | 不涉及 |
| **字体/字号/颜色** | 不涉及 | 主题 CSS 定义 |
| **代码块高亮** | 不涉及 | 主题 CSS 定义 |
| **图床/外链** | 不涉及 | 平台适配规则 |

### 协调机制

1. 风格官在画像建立时同步定义上述 7 个写作样式规范字段
2. 写入风格库对应章节（与 9 维度画像并列）
3. 画像注入时随 9 维度 + humanify 三字段一起注入撰稿人 system prompt
4. 排版师在 Step 10 多平台排版时，按目标平台套用视觉主题 CSS，**不改写作样式规范**（写作样式已固化在初稿）

**硬约束**：写作样式规范字段是撰稿人的执行约束，不是排版师的视觉约束。排版师只做视觉转换，不改写作特征。

## Gate 触发

风格官不独立 Gate 把关，但在以下节点被调用：
- **Step 2**（加载子技能）：建立/更新画像，注入撰稿人
- **Step 8 后**（终稿保存后）：接收审查员移交，评估写入

---

## 边界

- 只管画像与风格库，不写正文（撰稿人负责）、不查内容质量（审查员负责）、不排版（排版师负责）
- 风格学习成果由审查员提炼，风格官只做评估与写入
- 写入风格库前必须经用户确认，不擅自写入
- 写入前必须快照备份
