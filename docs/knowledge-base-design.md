# 领域知识库构建方案

## 核心理念

人类之所以能成为科研人员,是因为他看过大量领域教材、文献、案例,做过许多实验,积累了科研经验。这套知识库的设计,就是让多 Agent 团队也能累积这种"经验"。

不是给 Agent 塞一堆文件,而是建立一个**结构化、可检索、持续更新**的领域知识库,让每个 Agent 在干活时能像人类研究员"回忆起读过的某篇论文、某个实验方案、某个案例"那样,按需取用相关知识。

## 三层知识结构

对齐框架的"公司资料/项目资料/授权行业资料"三层,科研团队改造为:

### 1. 领域知识库(对应"授权行业资料")

团队级别的科研经验累积,所有项目共享:

| 子库 | 内容 | 谁负责更新 |
|---|---|---|
| **教材库** | 领域经典教材、综述、方法学教程(机器学习/统计/实验设计) | 资料员采集 + PI 筛选 |
| **论文库** | 已读论文的结构化登记(标题/作者/年份/DOI/分类标签/核心贡献) | 资料员采集 + 文献研究员登记 |
| **方法库** | 实验方案模板、统计方法选择指南、评估指标定义库 | 方法学专家沉淀 |
| **案例库** | 成功/失败的研究案例、被拒论文的审稿意见、复现报告 | 质检员 + 实现工程师沉淀 |
| **实验库** | 可复现实验包(代码版本+配置+数据+日志) | 实现工程师沉淀 |

### 2. 项目资料库(对应"项目资料")

单个研究项目专属:

| 内容 | 说明 |
|---|---|
| 项目底稿 | 意图/验收标准/边界/资料位置 |
| 研究蓝图 | PI 输出的子问题/假设/创新点/路线图 |
| 文献综述 | 文献研究员输出 |
| 实验方案 | 方法学专家输出 |
| 实验数据与日志 | 实现工程师输出(按引用,大文件不重复存) |
| 分析报告 | 数据分析师输出 |
| 论文初稿与版本 | 写作员输出 |
| 评审报告 | 质检员输出 |
| 合规报告 | 伦理员输出 |

### 3. 公司知识(对应"公司资料")

团队共享的规则与方法:

| 内容 | 说明 |
|---|---|
| 科研行为规范 | 引用规范、数据诚信、署名规则、LLM 使用声明 |
| 分类法标准 | 文献分类法的统一标准(由文献研究员维护) |
| 质量基线 | 质检标准、统计检验默认参数、可复现清单模板 |

## 知识库的"开放性"——不绑死后端

知识库通过**统一接口层**对接不同存储后端,用户可选:

| 后端 | 适合场景 | 接入方式 |
|---|---|---|
| **Obsidian**(本地 Markdown) | 个人研究者、本地优先、版本可控 | 文件系统读写,Markdown + frontmatter 元数据 |
| **Notion**(云端数据库) | 团队协作、可视化浏览、跨设备 | Notion API,数据库 + 页面 |
| **本地文件系统** | 简单、无依赖 | 目录结构 + YAML/JSON 索引 |
| **其他**(Zotero/EndNote 等) | 文献管理专业工具 | 通过其 API 或导出格式对接 |

### 统一接口契约

无论后端是什么,资料员和各 Agent 都通过统一接口操作知识库:

```
知识库操作:
  query(主题/关键词/分类) → 返回匹配条目列表
  get(引用键) → 返回单条详情
  add(条目) → 入库(自动去重)
  update(引用键, 字段) → 更新元数据
  list(分类/时间范围) → 列举
  export(格式) → 导出为 BibTeX/CSV/JSON
```

每个条目的标准 schema(对齐资料员交接契约):
```yaml
标题: ...
作者列表: [...]
年份: ...
venue: ...
DOI: ...
arXiv_ID: ...
分类标签: [...]
核心贡献: 一句话
摘要: ...(若有)
全文引用路径: ...(按引用取回,不重复存大文件)
入库时间: ...
采集信源: ...
验证状态: 已验证/待验证/未验证
```

## 资料员的定时采集机制

资料员配合框架的 Schedule(定时任务)实现信源自动采集:

### 信源清单配置示例

```yaml
# knowledge-sources.yaml
sources:
  - name: arxiv-cs-ai
    type: arxiv-api
    categories: [cs.AI, cs.CL, cs.LG]
    keywords: ["memory framework", "long-term memory", "agent memory"]
    frequency: daily
    api_endpoint: http://export.arxiv.org/api/query
    
  - name: semantic-scholar
    type: semantic-scholar-api
    keywords: ["LLM memory", "memory augmentation"]
    fields_of_study: [Computer Science]
    frequency: weekly
    api_endpoint: https://api.semanticscholar.org/graph/v1
    
  - name: openalex
    type: openalex-api
    concepts: [Memory architecture, Large language models]
    frequency: weekly
    
  - name: target-venues-rss
    type: rss
    feeds:
      - https://openreview.net/group?id=ICLR.cc/2026/Conference
      - https://openreview.net/group?id=NeurIPS.cc/2025/Conference
    frequency: daily
    
  - name: key-authors
    type: author-pages
    authors: ["作者名1", "作者名2"]
    frequency: monthly
```

### 定时任务配置(配合框架 Schedule)

资料员的 config.yaml 已设 `schedule: daily`,具体通过框架 Schedule 工具创建:

- **每日采集**(默认):扫 arXiv 新论文、目标会议 RSS
- **每周采集**:Semantic Scholar / OpenAlex 深度检索
- **每月采集**:关键作者主页、领域综述更新

每次采集完成后,资料员向秘书发采集报告,向活跃项目的文献研究员发"新到文献通知"。

## 让 Agent "积累科研经验"的机制

这是整个设计的核心——知识库不只是存储,要让 Agent 能"用到"积累的经验:

### 1. 按需检索,而非全量加载

Agent 不需要把知识库全量读进上下文。每个 Agent 在干活时,按当前需要检索相关知识:

| Agent | 检索什么 |
|---|---|
| PI | 同领域的已有研究空白、类似问题的路线图案例 |
| 文献研究员 | 已入库的论文分类法、已读论文清单(避免重复) |
| 方法学专家 | 方法库里的实验方案模板、统计方法选择指南 |
| 实现工程师 | 实验库里的可复现包、类似实验的实现案例 |
| 写作员 | 案例库里的同领域论文结构范例 |
| 质检员 | 案例库里的常见拒稿原因、审稿意见范例 |

### 2. 沉淀机制——每次研究的经验回流入库

研究不是一次性的,每次项目的产出要沉淀回领域知识库:

| 产出 | 沉淀到 | 由谁沉淀 |
|---|---|---|
| 新读的论文 | 论文库 | 文献研究员 |
| 新的实验方案 | 方法库 | 方法学专家 |
| 实验代码包 | 实验库 | 实现工程师 |
| 审稿意见/被拒原因 | 案例库 | 质检员 |
| 复现报告 | 案例库 | 实现工程师/数据分析师 |

### 3. 分类法持续演化

文献研究员维护的分类法不是固定的,随研究深入持续演化:
- 新方向出现 → 加分类标签
- 旧分类合并 → 调整标签
- 资料员按最新分类法给新入库文献打标

## 知识库冷启动

新团队的知识库是空的,按以下顺序冷启动:

1. **PI 定领域方向** — 确定要研究什么(如 AI 记忆框架)
2. **资料员批量采集** — 按方向配置信源,跑一轮批量采集(过去 2-3 年的核心论文)
3. **文献研究员建分类法** — 基于首批论文建初始分类法
4. **资料员按分类法回溯打标** — 给已入库的论文补分类标签
5. **PI 基于初始库定创新点** — 此时信息足够,PI 可锁定方向
6. **后续持续采集** — 资料员定时采集,知识库持续增长

## 与框架现有机制的对接

| 框架机制 | 科研团队对接 |
|---|---|
| 公司/项目/授权行业三层资料 | 改造为领域知识库/项目资料/公司知识三层 |
| 资料库权限(只读/可写/可管) | 领域知识库全员只读,资料员可写,分类法文献研究员可管 |
| 按引用取回大文件 | 论文全文、实验数据按引用,不重复传 |
| Schedule 定时任务 | 驱动资料员定时采集 |
| 11 类硬管控动作 | "记入长期记忆""晋升为公司知识"对应知识库沉淀 |
| 带版本工作包交接 | 各 Agent 交接时引用知识库条目,不传全文 |
