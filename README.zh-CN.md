# Digital Office 数字办公系统

Digital Office 是一套面向中小企业的多 Agent 数字员工团队。它把秘书、研究、产品、规划、设计、开发、写作和法务等角色装进同一套可部署系统里，让企业可以用一个入口交办任务，再由秘书完成意图澄清、角色编排、进度跟踪、审批和交付记录。

## 适合谁

- 创业公司和小团队：人手少，但需要有人接住调研、文档、项目推进和交付检查。
- 中小企业部门：需要把分散的 AI 使用方式收敛成可审计、可复盘、可控成本的团队流程。
- 产品和研发团队：希望把需求、设计、开发、测试和发布质量门禁串成一条可运行的链路。
- 内容和知识团队：需要稳定的写作、资料整理、事实核对和知识沉淀流程。

## 核心能力

- 一个秘书入口：用户只描述目标，秘书负责确认意图、选择角色、拆分工作和协调交接。
- 多 Agent 编排：每件事有明确负责人，必要时串联研究、产品、设计、开发、写作和法务角色。
- Skill 优先执行：Agent 通过内置 Skill 和生产 harness 完成任务，而不是只依赖自由发挥的提示词。
- 长程任务控制：Loop 有迭代次数、预算、重试、审批、暂停、恢复和完成状态，不允许无限自转。
- 可审计交接：跨角色协作使用带版本的工作包，记录目标、事实、假设、验收标准、风险和产物引用。
- 企业安全边界：删除、覆盖、外发、发布、付款、数据导出、长期记忆写入和专业建议等高风险动作需要审批。

## 部署与运行

推荐在 WSL 或 Linux 环境安装到 Hermes/OpenClaw 主目录：

```bash
curl -fsSL https://raw.githubusercontent.com/digibeing1001/digital-office-agent-system/main/install.sh | bash
```

默认安装到 `~/.hermes`。安装器会注入默认秘书 Agent 入口，并在已有 `SOUL.md` 或 `AGENTS.md` 时要求选择保留并行安装或明确覆盖，避免误删企业本地规则和个人资料。

启动图形界面：

```bash
digital-office-gui
digital-office-gui --admin
```

也可以使用 Web/PWA 命令：

```bash
web-config
web-serve
```

普通用户入口是 `/`，管理后台入口是 `/admin`。PWA 会注册 `manifest.webmanifest` 和 `service-worker.js`，适合内网、小服务器和单机部署。

## 企业使用要求

- 上线前运行 `agent-system/bin/harness-check` 和核心 production harness。
- 先配置公司知识库、项目知识库和权限边界，再开放给员工使用。
- 对法务、合规、财务、人事等受监管输出，系统只提供内部审查草稿，最终判断必须由企业负责人或专业人员确认。
- 升级时使用仓库内 `update` 脚本；运行数据、项目、审批、个人偏好、模型密钥和 profile 私有状态不应被覆盖。

## 三个分支

| 分支 | 定位 | 场景 |
| --- | --- | --- |
| `main` | 通用数字办公系统 | 中小企业通用办公、产品、设计、研发、写作和法务 |
| `research-team` | 科研团队特化 | 文献、方法、数据、复现、伦理、论文写作和同行评审 |
| `writer-team` | 写作团队特化 | 选题、采访、起草、编辑、事实核对、风格润色和发布准备 |
