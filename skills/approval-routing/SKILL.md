# approval-routing — 审批路由

## 用途
识别科研流程中的硬管控动作,无条件阻断并路由到对应审批方(人工 / 专业复核角色)。与风险评分不同,硬管控不看风险分,只要触发就阻断,确保关键节点不绕过人审。

适用于:科研秘书对所有"不可单方面执行"动作的把关。

## 触发条件
- 任何角色准备执行下列硬管控动作之一
- 系统检测到动作类型属于 11 类硬管控清单
- 用户直接要求跳过某个评审/审批环节

## 操作步骤

### 步骤 1:识别动作类型
检查待执行动作是否属于以下 11 类硬管控:
1. 实验方案定稿(方法学专家产出)
2. 论文投稿提交
3. 申报书提交
4. 数据集对外发布
5. 代码仓库公开(public)
6. 涉及人类受试者的实验启动
7. 涉及动物实验的方案
8. 删除/覆盖已有实验记录
9. 大额采购(算力 / 设备 / 数据)
10. 与外部合作方共享数据
11. 撤回已发表的论文 / 撤回已提交的申报

### 步骤 2:匹配硬管控清单
- 命中任一类即进入阻断流程
- 不在清单内的动作走风险评分技能,不走本技能

### 步骤 3:阻断
- 立即停止动作执行
- 不得以"风险分低"为由跳过阻断
- 不得以"用户催"为由放行

### 步骤 4:路由
按动作类型路由到对应审批方:
- **人工审批**:用户本人或用户指定的负责人(适用于 1/2/3/5/10/11)
- **专业复核**:对应专业角色(方法学专家审 6/7;PI 审 4;采购流程审 9)
- **双重审批**:既需人工又需专业复核(适用于 6/7 涉伦理的)

### 步骤 5:记录审批结果
按输出格式记录审批全流程,留痕备查。

## 调用示例
```
触发:学术写作员准备点击论文投稿按钮
识别:动作类型 = 论文投稿提交(第 2 类)
阻断:停止提交
路由:人工审批(用户本人) + 专业复核(PI 确认 claim 完整性)
记录:审批单 #20260701-001
```

## 输出格式
审批记录(写入 `outputs/approval/<date>-<seq>.md`):
- **审批单号**:日期-序号
- **动作类型**:11 类之一
- **触发来源**:哪个角色 / 哪个流程
- **路由去向**:人工 / 专业复核 / 双重
- **审批方**:具体人或角色名
- **审批结果**:通过 / 驳回 / 待定
- **审批意见**:审批方的具体意见
- **时间戳**:阻断时间 + 审批完成时间
- **处置**:放行执行 / 返工 / 终止

## 约束
- 硬管控无条件阻断,不靠风险分,不靠用户口头同意
- 11 类清单是封闭集合,不得自行增删(增删需走系统配置变更)
- 审批必须留痕,审批单号唯一可追溯
- 审批方未明确前不得放行
- 用户要求跳过审批时,必须明确告知"这是硬管控,不能跳过",并记录用户要求
- 双重审批两项都通过才能放行,任一驳回即终止
