# innovation-positioning — 创新点定位

## 用途
对比现有工作,定位本研究的创新空白,产出有证据支撑的创新点定位报告。禁止自评"创新性强",必须用文献证据说话。

适用于:研究方案定型前、论文 Introduction 撰写前、申报书创新点章节撰写前。

## 触发条件
- 子问题和假设构建完成,需要明确本研究"新在哪"
- 论文 Introduction 第 3 段(贡献段)撰写前
- 申报书"创新点"章节撰写前
- 用户问"我们这个研究到底新在哪"

## 操作步骤

### 步骤 1:列出本研究方案
1. 从子问题和假设清单读取本研究要做的事
2. 从方案设计读取方法/数据/应用三个维度的具体内容
3. 整理成"我们做了什么"清单(3-5 条)

### 步骤 2:检索已有方案
1. 调用文献检索技能,搜索与本研究最相关的现有工作(至少 10 篇近 3 年文献)
2. 按创新类型分组:
   - 方法创新:用了不同的算法/架构/训练方式
   - 数据创新:用了新的数据集/数据来源/数据规模
   - 应用创新:应用到新领域/新场景/新人群
3. 列出每个已有方案做了什么、没做什么

### 步骤 3:用学术图找未覆盖区域
1. 构建"已有工作覆盖图":横轴=方法维度,纵轴=应用/数据维度
2. 标注每个已有工作覆盖的格子
3. 找空白格子(没人做过的组合)
4. 评估空白格子是否有价值(有研究意义 vs 没人做因为没意义)

### 步骤 4:定位创新点
针对每个候选创新点:
1. 明确创新类型(方法/数据/应用)
2. 与最接近的已有工作对比,说明差异
3. 给出文献证据(引用 + 该文献做了什么)
4. 评估创新强度:
   - 增量式:在已有方法上改进
   - 组合式:已有方法的新组合
   - 范式式:全新的问题形式化或方法范式(罕见)

### 步骤 5:检查多样性
1. 创新点不应全部集中在同一类型(全方法创新 or 全数据创新都不健康)
2. 至少有 1 个方法或应用层面的创新,纯数据创新容易被质疑
3. 创新点之间不重复,各自独立成立

## 调用示例
```
本研究:用 LoRA 微调 7B 模型做长文本任务
已有工作:
  - Smith 2023: 全参微调 13B 做长文本 [方法不同]
  - Wang 2024: LoRA 微调 7B 做短文本 [任务不同]
  - Li 2024: LoRA 微调 70B 做长文本 [规模不同]
空白:LoRA + 7B + 长文本组合没人系统做过
创新点:
  类型:组合式方法创新
  差异:首次系统评估中等规模模型 + LoRA + 长文本的可行性
  证据:[Smith 2023] [Wang 2024] [Li 2024] 均未覆盖该组合
  强度:增量式
```

## 输出格式
创新点定位报告(写入 `outputs/pi/<project>-innovation.md`):
- **本研究方案摘要**:3-5 条
- **已有工作清单**:每条含 文献引用 / 做了什么 / 没做什么
- **已有工作覆盖图**:方法 × 数据/应用 二维表
- **创新点清单**:每条含
  - 创新类型(方法/数据/应用)
  - 创新描述
  - 与最接近已有工作的差异
  - 文献证据(引用列表)
  - 创新强度(增量/组合/范式)
- **多样性检查**:各类创新点分布

## 约束
- 创新点必须查文献比对,禁止自评"创新性强"
- 每个创新点必须有至少 3 篇文献作为对比证据
- 不得把"我们用了更新的数据"当作方法创新
- 范式式创新需特别谨慎,必须有充分证据证明前人没做过
- 创新点数量控制在 2-4 个,过多说明没抓重点
- 创新强度诚实评估,不夸大增量式为范式式
