# taxonomy-building — 分类法建设

## 用途
为纳入的文献建立分类法,按方法/任务/数据集/应用领域等维度组织文献,使后续综述、研究空白识别、引用整合有统一的索引框架。分类法由文献研究员建设,资料员对齐打标签。

适用于:PRISMA 筛选完成后,综述撰写前。

## 触发条件
- PRISMA 筛选完成,有纳入文献清单
- 用户要求"给文献分类""建分类法"
- 已有分类法需要修订或扩展

## 操作步骤

### 步骤 1:扫描纳入文献
1. 读取 PRISMA 纳入文献清单
2. 逐篇提取关键信息:
   - 研究方法(用了什么算法/技术)
   - 研究任务(解决什么问题)
   - 数据集(用了什么数据)
   - 应用领域(在哪个场景)
   - 评估指标(用什么衡量效果)
3. 汇总所有提取的维度值

### 步骤 2:提取共同维度
1. 找出大多数文献都涉及的维度(覆盖率 > 60%)
2. 候选维度:方法 / 任务 / 数据集 / 应用领域 / 评估指标 / 模型规模 / 时间
3. 选取 2-4 个最有区分度的维度作为分类轴
4. 与 PI 确认维度选择

### 步骤 3:建分类法(一级/二级)
对每个选定的维度:
1. **一级分类**:粗粒度,5-8 类
   - 方法维度示例:监督学习 / 自监督 / 强化学习 / 混合方法
   - 任务维度示例:分类 / 生成 / 检索 / 排序
2. **二级分类**:细粒度,每类下 3-5 子类
   - 监督学习下:全参微调 / 参数高效微调 / 提示学习
3. 每个分类节点给定义(什么算这一类),避免主观
4. 标注分类间的互斥/包含关系

### 步骤 4:标注每篇文献的分类
1. 对每篇纳入文献,按分类法的每个维度打标签
2. 一篇文献在一个维度下可打多个标签(如同时用了监督和自监督)
3. 标注时引用文献里的具体证据(哪段哪节说明用了这个方法)
4. 无法归类的文献放入"其他",并分析是否需要新增分类

### 步骤 5:回填已有库
1. 把分类法和标签写入文献库(BibTeX + 标签字段)
2. 与资料员对齐标签命名(资料员负责打标签,文献研究员负责定标准)
3. 输出分类法定义 + 文献标签表

## 调用示例
```
纳入文献 55 篇,提取维度:方法 / 任务 / 数据集
分类法定义:
  方法(一级):监督 / 自监督 / 强化 / 混合
    监督(二级):全参微调 / LoRA / Prefix-tuning / Prompt
  任务(一级):分类 / 生成 / 检索
  数据集(一级):通用 / 领域 / 自建
文献 [Smith 2023] 标签:
  方法:监督.全参微调
  任务:分类
  数据集:通用
  证据:Section 3.2 描述全参微调,Section 4 用 GLUE
```

## 输出格式
- **分类法定义**(写入 `outputs/lit/<project>-taxonomy.md`):
  - 维度清单(2-4 个)
  - 每个维度的一级分类(含定义)
  - 每个一级分类下的二级分类(含定义)
  - 分类间关系(互斥/包含)
- **文献分类标注表**(写入 `outputs/lit/<project>-labels.md`):
  - 文献标识(DOI/标题)
  - 各维度标签
  - 证据引用(哪段说明)
- **未归类文献清单**:附分析(是否需新增分类)

## 约束
- 分类法由文献研究员建,资料员对齐打标签,角色不混
- 一级分类 5-8 类,二级分类每类 3-5 子类,过多说明粒度太细
- 每个分类必须有定义,不能只给名字
- 文献打标签必须附证据(文献里哪段说明),不得主观判断
- 分类法应覆盖 ≥ 90% 的纳入文献,大量"其他"说明分类法有问题
- 分类法建成后在项目内固定,如需修订走变更流程
