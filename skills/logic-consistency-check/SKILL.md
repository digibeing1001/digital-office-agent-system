# logic-consistency-check — 逻辑一致性检查

## 用途
检查论文逻辑链条是否连贯,提取论证链条(前提→推理→结论),检查每步推理、前后矛盾和断点,确保论文叙事自洽、方法与实验对应、图表与正文一致。

适用于:论文逻辑审查、各节叙事一致性检查、图表正文核对等场景。

## 触发条件
- 用户要求"检查逻辑""逻辑一致性""叙事连贯"时。
- 论文各章节之间需要核对是否自洽时。
- 用户提到"逻辑""矛盾""前后不一致"时。

## 操作步骤
1. **提取论证链条**:梳理全文论证结构(引言提出问题→方法解决问题→实验评估→结论总结),画出前提→推理→结论的链条。
2. **检查每步推理**:检查每步推理是否成立,前提是否支撑结论,有无跳跃。
3. **检查前后矛盾**:
   - 摘要 vs 结论:核心结论是否一致。
   - 方法描述 vs 实验设置:方法说的组件是否都在实验中出现,实验用的设置是否都在方法中描述。
   - 正文数字 vs 图表数字:正文说"提升 5.2%",图表是不是 5.2%。
   - 术语:同一概念是否全文统一命名。
4. **标记断点**:对论证链条中断、推理跳跃、前后矛盾的位置逐个标记,标明章节和段落。

## 调用示例
```
输入:论文草稿
输出:逻辑一致性报告
```
```markdown
## 矛盾 1
- 位置:摘要 vs 第 6 节结论
- 问题:摘要说"在所有任务上提升",结论说"在多数任务上提升"
- 建议:统一表述,以实验证据为准。

## 断点 1
- 位置:第 3.2 节 → 第 4.1 节
- 问题:方法介绍了模块 X,但实验未对 X 做 ablation
- 建议:补充 X 的 ablation 或说明原因。
```

## 输出格式
逻辑一致性报告,含:
- 论证链条图(前提→推理→结论)
- 矛盾清单(位置 + 问题描述 + 建议)
- 断点清单(位置 + 缺失环节 + 建议)
- 术语一致性检查结果

## 约束
- **标记具体位置**:每处问题必须标到章节/段落/图表,不泛泛说"逻辑不顺"。
- 检查覆盖全文,不只看单节。
- 数字核对必须逐个比对正文和图表。
- 不替作者改写,只指出问题。
