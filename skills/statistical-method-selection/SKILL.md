# statistical-method-selection — 统计方法选择

## 用途
根据研究问题的类型和数据特征,选择合适的统计方法,明确前提假设,给出选择理由。确保方法匹配数据类型和问题类型,避免"看到数据就跑回归"的误用。

适用于:实验方案设计阶段,统计方法选型;结果分析阶段,确认方法是否合适。

## 触发条件
- PI 产出子问题和假设,需要选统计方法
- 实验方案设计阶段
- 用户问"该用什么统计方法""用什么检验"
- 已有分析方法但需要核查是否合适

## 操作步骤

### 步骤 1:识别数据类型
检查实验数据的数据类型:
- **类别型(categorical)**:名义(无序,如性别)/ 序数(有序,如等级)
- **连续型(continuous)**:区间(如温度)/ 比率(如长度)
- **计数型(count)**:离散非负整数(如点击次数)
- **时间型(time-to-event)**:生存数据
- **多变量(multivariate)**:多个因变量同时分析

### 步骤 2:识别研究问题类型
从子问题和假设识别问题类型:
- **比较(comparison)**:两组或多组是否有差异
  - 示例:LoRA vs 全参性能差异是否显著
- **关联(association)**:变量间是否相关
  - 示例:模型规模与性能是否相关
- **预测(prediction)**:用一些变量预测另一个
  - 示例:用训练损失预测最终性能
- **因果(causation)**:一个变量是否导致另一个
  - 示例:微调方式是否导致性能差异
- **聚类(clustering)**:数据是否能自然分组

### 步骤 3:选统计方法
按"数据类型 × 问题类型"矩阵选方法:

| 问题类型 | 类别型因变量 | 连续型因变量 | 计数型因变量 |
|----------|------------|------------|------------|
| 比较(2组) | 卡方 / Fisher | t 检验 / Mann-Whitney | 卡方 / 泊松检验 |
| 比较(>2组) | 卡方 | ANOVA / Kruskal-Wallis | 卡方 |
| 关联 | 卡方 / Cramér's V | Pearson / Spearman 相关 | — |
| 预测 | 逻辑回归 / 分类模型 | 线性回归 / 回归模型 | 泊松回归 |
| 因果 | — | 实验设计 + ANCOVA | — |

考虑因素:
- 样本量大小(小样本用非参数方法)
- 分布特征(非正态用非参数)
- 重复测量(用重复测量 ANOVA / 混合模型)
- 多重比较(用 Bonferroni / FDR 校正)

### 步骤 4:检查前提假设
对选定的方法,逐项检查前提假设:
- **t 检验 / ANOVA**:正态性(Shapiro-Wilk)+ 方差齐性(Levene)+ 独立性
- **线性回归**:线性关系 + 残差正态 + 同方差性 + 独立性 + 无多重共线
- **逻辑回归**:线性(对数比)+ 无完全分离 + 大样本
- **非参数方法**:无需分布假设,但要求独立同分布
- **卡方**:期望频数 ≥ 5(每格)

前提不满足的处理:
- 换方法(如 t 检验不满足正态换 Mann-Whitney)
- 数据变换(如对数变换)
- 用稳健方法(如 bootstrap)

### 步骤 5:输出选择理由
按输出格式产出方法选择报告,包含选择逻辑链。

## 调用示例
```
研究问题:LoRA 与全参微调在 LongBench 上的性能差异是否显著
数据类型:连续型(准确率),两组独立
问题类型:比较(2组)
候选方法:t 检验(参数)/ Mann-Whitney U(非参数)
前提检查:
  正态性:Shapiro-Wilk p=0.12 → 满足
  方差齐性:Levene p=0.45 → 满足
  独立性:不同随机种子独立运行 → 满足
选择:独立样本 t 检验
多重比较:LongBench 有多个子任务 → Bonferroni 校正
理由:数据连续 + 两组独立 + 满足正态和方差齐性 + 多任务需校正
```

## 输出格式
统计方法选择报告(写入 `outputs/method/<project>-stats.md`):
- **研究问题**(对应子问题编号)
- **数据类型**:类别/连续/计数/时间/多变量
- **问题类型**:比较/关联/预测/因果/聚类
- **候选方法**:2-3 个
- **前提假设检查**:每项 通过/未通过 + 检验结果
- **选择方法**:最终选定的方法
- **多重比较校正**:是否需要 + 校正方法
- **选择理由**:逻辑链(数据类型+问题类型+前提满足→方法)
- **备选方案**:前提不满足时的替代方法

## 约束
- 方法必须匹配数据类型和问题类型,不得"看到数据就跑回归"
- 前提假设必须显式检查,不得默认满足
- 前提不满足时换方法或用稳健方法,不得硬上
- 多重比较必须校正(Bonferroni / FDR / Holm)
- 不替 PI 定研究问题,只针对已定问题选方法
- 选择理由必须给出逻辑链,不得只说"用 t 检验"
- 涉及因果推断必须明确实验设计(RCT / 准实验 / 观察性)
