# gap-identification — 研究空白识别

## 用途
从文献综述和分类法中识别研究空白(未被现有工作覆盖的区域),评估每个空白的研究价值,为 PI 的创新点定位和后续研究方向选择提供依据。

适用于:文献综述和分类法完成后,创新点定位前。

## 触发条件
- taxonomy-building 产出分类法
- 文献综述撰写完成
- 用户要求"找研究空白""哪里还没人做"
- PI 需要为创新点定位提供输入

## 操作步骤

### 步骤 1:分析分类法
1. 读取分类法定义和文献标签表
2. 构建覆盖矩阵:维度1 × 维度2,每个格子标注有多少篇文献覆盖
3. 识别空格子(0 篇)和稀疏格子(1-2 篇)
4. 识别密集格子(≥ 10 篇,说明该方向已饱和)

### 步骤 2:识别未覆盖区域
对每个空/稀疏格子:
1. 检查是否真的没人做,还是搜索时漏了(补检索确认)
2. 如果确实没人做,分析原因:
   - 没意义:技术上行不通或无应用价值 → 跳过
   - 没条件:数据/算力/技术不可得 → 标注障碍
   - 没想到:有潜力但尚未被探索 → 候选空白
3. 如果是稀疏格子,分析现有工作的不足:
   - 方法局限:用了次优方法
   - 数据局限:数据规模/质量不够
   - 评估局限:指标/基准不全

### 步骤 3:对照已有方案
对每个候选空白:
1. 找到最接近的已有工作(至少 3 篇)
2. 分析已有工作做了什么、没做什么
3. 明确空白的边界:不是"完全没人做",而是"这个具体角度没人系统做过"

### 步骤 4:定位空白
把候选空白归类:
- **方法空白**:某类方法没被应用到某任务
- **数据空白**:某数据规模/类型/领域没被覆盖
- **任务空白**:某任务/场景没被研究
- **评估空白:某评估维度/指标没被考虑
- **规模空白**:某规模区间没被验证(如 7B 模型 + 长文本)
- **组合空白**:多个维度的组合没人系统研究

### 步骤 5:评估空白价值
对每个空白评估:
1. **研究意义**:解决了能回答什么问题、推动什么进展(高/中/低)
2. **可行性**:数据/算力/技术是否就位(高/中/低)
3. **时效性**:是否紧迫(别人可能在做)(高/中/低)
4. **创新强度**:增量 / 组合 / 范式
5. 综合排序:高价值空白优先

## 调用示例
```
分类法覆盖矩阵(方法 × 任务):
              分类    生成    检索
  监督学习     12篇    8篇     5篇
  自监督       3篇     0篇★    1篇
  强化学习     0篇★    0篇★    0篇★
  混合方法     2篇     1篇     0篇★
候选空白:
  自监督 + 生成(0篇):有潜力,数据可获取 → 高价值
  强化学习 + 检索(0篇):技术不可行(RL reward 难定义)→ 跳过
已有工作对比:自监督生成最近有 SimSemantic 等探索,但未在 X 任务上系统验证
```

## 输出格式
研究空白清单(写入 `outputs/lit/<project>-gaps.md`):
- **分类法覆盖矩阵**:维度1 × 维度2 的文献数分布
- **空白清单**:每条含
  - 空白编号
  - 空白描述(哪个维度的什么组合没人做)
  - 空白类型(方法/数据/任务/评估/规模/组合)
  - 现有覆盖情况(最接近的已有工作 + 它们做了什么)
  - 空白原因(没意义/没条件/没想到)
  - 价值评估(研究意义/可行性/时效性/创新强度)
  - 综合排序(高/中/低价值)
- **已排除空白**:附排除理由(没意义/不可行)

## 约束
- 空白必须基于文献证据(分类法覆盖矩阵),不得主观臆断"没人做"
- 空白描述要具体(哪个维度的什么组合),不得笼统说"X 领域研究不足"
- 必须对照已有方案,明确"完全没人做"vs"这个角度没人系统做"
- 评估空白价值要诚实,没意义的空白要排除,不要硬凑
- 空白数量控制在 3-7 个,过多说明分类法粒度太细
- 高价值空白优先,低价值的可以列入但不作为创新点候选
