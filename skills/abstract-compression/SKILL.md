# abstract-compression — 摘要压缩

## 用途
从论文正文压缩出符合规范的摘要,150-250 字,包含问题/方法/结果/贡献四要素,与 Introduction 和 Conclusion 呼应但不雷同。

适用于:论文/报告正文完成后,摘要撰写环节。摘要是论文最后写的部分。

## 触发条件
- 论文正文全部完成
- 用户要求"写摘要""压缩摘要"
- 摘要需要按字数限制调整
- 投稿前检查摘要完整性

## 操作步骤

### 步骤 1:提取核心四要素
从正文提取:
1. **问题(Problem)**:Introduction 第 1-2 段,1-2 句
   - 研究的大背景 + 具体问题
2. **方法(Method)**:Method 章节核心,1-2 句
   - 用了什么方法 + 关键创新
3. **结果(Results)**:Experiments 主结果,1-2 句
   - 关键数字(必须可量化)
4. **贡献(Contribution)**:Introduction 贡献段或 Conclusion,1 句
   - 本研究的意义或影响

### 步骤 2:按摘要结构组织
标准摘要结构(可微调):
1. 背景句(1-2 句):领域重要性 + 现有不足
2. 问题句(1 句):本文要解决的具体问题
3. 方法句(1-2 句):本文做了什么 + 怎么做的
4. 结果句(1-2 句):关键发现 + 数字
5. 意义句(1 句):为什么重要

示例骨架:
"[背景] X 是重要问题。 [问题] 然而,现有方法在 Y 上表现不佳。 [方法] 本文提出 Z 方法,通过 W 机制解决该问题。 [结果] 在 D 数据集上,Z 方法比 SOTA 提升 N%,且开销更低。 [意义] 这表明 W 机制对 X 类问题具有普适意义。"

### 步骤 3:压缩到字数限制
1. 确认目标字数限制:
   - NeurIPS:通常 250 字以内
   - ICML:通常 250 字以内
   - ICLR:通常 250 字以内
   - ACL:长摘要 250 字 / 短摘要 125 字
2. 压缩策略:
   - 删除冗余背景(读者已知的不用说)
   - 合并相似表述(背景+问题合并)
   - 用数字替代描述("提升 5.2%"代替"有显著提升")
   - 删除细节(具体超参不进摘要)
3. 压缩后朗读,检查是否"听起来像摘要"

### 步骤 4:检查完整性
四要素检查:
1. **问题**:读者能看出在解决什么问题吗?
2. **方法**:读者能看出用了什么方法吗?
3. **结果**:有可量化的关键数字吗?
4. **贡献**:读者能看出这件事的意义吗?
任一缺失,补上。

### 步骤 5:呼应检查
1. 摘要的 claim ⊆ Introduction 的 claim ⊆ Conclusion 的 claim
2. 摘要的数字 = 正文表格的数字 = 图表 caption 的数字
3. 摘要不得引入正文没有的新信息
4. 摘要不得引用文献、不得引用图表编号
5. 摘要与 Introduction 第 1 段和 Conclusion 第 1 段呼应但不雷同(不复制粘贴)

## 调用示例
```
正文核心:
  问题:LoRA 在长文本上效果不明
  方法:系统评估 LoRA 在 7B 模型 + 长文本任务上的表现
  结果:LongBench 上比全参低 2.1%,但训练成本降 70%
  贡献:首次系统验证 LoRA 在中等规模长文本场景的可行性

摘要草稿:
  大型语言模型在长文本任务上微调成本高昂。LoRA 通过低秩
  分解减少可训练参数,但其在长文本任务上的效果尚不明确。
  本文系统评估 LoRA 在 7B 模型和 LongBench 基准上的表现。
  实验表明,LoRA(r=8)在 LongBench 上平均分比全参微调低
  仅 2.1%,但训练成本降低 70%。这一结果表明 LoRA 在中等
  规模长文本场景下具有良好的性价比,为资源受限场景提供了
  可行方案。

完整性:问题✓ 方法✓ 结果✓(有数字) 贡献✓
字数:178 字 ✓
```

## 输出格式
摘要文本(写入 `outputs/writing/<project>-abstract.md`):
- **摘要正文**
- **字数**
- **四要素检查**:问题/方法/结果/贡献 各项通过状态
- **呼应检查**:与 Intro/Conclusion 的一致性
- **压缩说明**:从多少字压缩到多少字,删了什么

## 约束
- 摘要必须包含问题/方法/结果/贡献四要素,缺一不可
- 结果必须有可量化数字,禁止"有显著提升"这类表述
- 摘要不引用文献、不引用图表编号
- 摘要与 Intro/Conclusion 呼应但不雷同,不复制粘贴
- 摘要不得引入正文没有的新信息
- 字数严格遵循会议要求,超了必须压缩
- 摘要是论文最后写的部分,正文未定稿不写摘要
