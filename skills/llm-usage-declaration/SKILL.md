# llm-usage-declaration — LLM使用声明

## 用途
识别论文写作和研究过程中哪些环节用了 LLM,评估使用程度,按目标期刊/会议的 LLM 使用政策生成声明文本,并建议放在论文的哪个位置。

适用于:论文 LLM 使用声明生成、AI 工具使用披露、期刊政策合规等场景。

## 触发条件
- 用户要求"生成 LLM 声明""LLM 使用声明""AI 使用披露"时。
- 论文写作中用了 LLM 需要声明时。
- 用户提到"LLM 声明""AI 使用""ChatGPT 声明"时。

## 操作步骤
1. **识别哪些环节用了 LLM**:梳理研究全流程(文献调研 / 方法设计 / 代码编写 / 数据分析 / 论文写作 / 翻译润色),标注每环节是否用了 LLM。
2. **评估使用程度**:对每个用了 LLM 的环节,评估使用程度(辅助 / 部分 / 主要),记录所用模型(如 GPT-4 / Claude)。
3. **按期刊要求生成声明**:查目标期刊/会议的 LLM 使用政策,按其要求格式生成声明文本。
4. **建议放在论文哪里**:按期刊要求建议放置位置(致谢 / 方法 / 声明页 / 脚注)。

## 调用示例
```
输入:LLM 使用情况(环节/模型/程度)+ 目标期刊
输出:LLM 使用声明文本 + 放置建议
```
```markdown
## LLM 使用情况
- 文献调研:用 GPT-4 辅助摘要筛选(辅助)
- 代码编写:未使用
- 论文写作:用 Claude 辅助润色英文(辅助)
- 翻译:未使用

## 声明文本(按 Nature 政策)
"在撰写本论文时,作者使用了 GPT-4(文献调研)和
Claude(英文润色)作为辅助工具。所有研究内容、分析和
结论均由作者独立完成并负责。"

## 放置建议:致谢章节末尾或声明页
```

## 输出格式
LLM 使用声明,含:
- LLM 使用情况清单(环节 / 模型 / 使用程度)
- 声明文本(按目标期刊/会议政策格式)
- 放置位置建议

## 约束
- **按目标期刊/会议的 LLM 使用政策**:不同期刊要求不同,必须按目标政策生成。
- 如实声明,不得隐瞒 LLM 使用。
- LLM 不得列为作者(按 ICMJE / Nature 等政策)。
- LLM 生成的内容需作者核实,声明中注明作者对全文负责。
