# methodology-critique — 方法学批评

## 用途
审查论文方法学是否严谨,从实验设计、统计方法、baseline 公平性、可复现性四个方面做批评,产出具体到可改的问题清单,避免"方法学没问题"的空泛结论。

适用于:方法学审查、实验严谨性评估、投稿前方法学自检等场景。

## 触发条件
- 用户要求"方法学审查""方法学批评""检查实验严谨性"时。
- 需要评估实验设计和统计方法是否站得住脚时。
- 用户提到"methodology""方法学""实验严谨"时。

## 操作步骤
1. **检查实验设计**:对照 experiment-design 的清单,看评估维度是否完备、ablation 是否有假设、样本量是否充分。
2. **检查统计方法**:看检验方法是否匹配数据分布、是否报告效应量、多重比较是否校正、是否只报 p 值。
3. **检查 baseline 公平性**:数据集是否相同、超参搜索预算是否相同、计算预算是否可比、评估协议是否一致。
4. **检查可复现性**:随机种子、软件版本、超参、数据 split、硬件是否完整记录。
5. **输出方法学批评**:把发现的问题列成清单,每条标严重度(致命/重大/中等/小),给具体修改建议。

## 调用示例
```
输入:论文草稿(方法+实验章节)
输出:方法学批评报告
```
```markdown
## 重大问题
1. [统计] 未报告效应量,只报 p 值(第 5.3 节)
   建议:补充 Cohen's d 和 95% 置信区间。
2. [公平性] baseline X 用了弱配置(表 2 注释)
   建议:用 baseline X 官方推荐超参重跑。

## 中等问题
3. [ablation] 去模块 Y 的 ablation 无明确假设(第 4.4 节)
   建议:补充"若 Y 有效则去除后指标应下降"的假设陈述。
```

## 输出格式
方法学批评报告,含:
- 问题清单(按严重度排序:致命/重大/中等/小)
- 每条:问题位置(章节/图表)+ 问题描述 + 修改建议
- 总体方法学评价(严谨/基本严谨/有缺陷/不严谨)

## 约束
- **批评必须具体到可改**:不说"实验不充分",要说"第 5.3 节未报告效应量,建议补充"。
- 不泛泛而谈,每条批评必须有论文具体位置。
- 严重度分级要公正,不夸大也不压低。
- 不替作者改写,只指出问题给建议。
