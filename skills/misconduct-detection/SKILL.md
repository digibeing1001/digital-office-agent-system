# misconduct-detection — 学术不端检测

## 用途
接收质检员标记的疑似学术不端信号,用工具(JPlag / MinHash / ELA)做深入核查,区分无意错误和故意不端,生成学术不端检测报告,标注"疑似"而不单独判定造假。

适用于:学术不端排查、造假/篡改/剽窃检测、疑似信号深入核查等场景。

## 触发条件
- 质检员标记疑似学术不端信号时。
- 用户要求"检测学术不端""核查造假""查篡改"时。
- 用户提到"学术不端""misconduct""造假"时。

## 操作步骤
1. **接收质检员标记的疑似信号**:从 peer-review / methodology-critique 等技能获取疑似信号(数字异常 / 文本雷同 / 图片篡改等)。
2. **用工具深入核查**:
   - 剽窃:用 JPlag(代码)+ MinHash(文本)查重(复用 plagiarism-check)。
   - 图片篡改:用 ELA(Error Level Analysis)分析图片是否后期编辑。
   - 数字异常:用统计方法检查数字是否异常完美(见 outlier-detection)。
3. **区分无意错误和故意不端**:判断是偶发错误(笔误 / 计算失误)还是系统性不端(选择性报告 / 数据捏造 / 篡改)。
4. **生成报告**:汇总核查结果,每条标严重度和核查方法,标注"疑似"而非"确认"。

## 调用示例
```
输入:质检员标记的疑似信号 + 论文草稿 + 原始数据/图片
输出:学术不端检测报告
```
```markdown
## 疑似信号 1(来自 peer-review):数字异常完美
- 核查方法:统计检验(outlier-detection)
- 结果:所有指标恰好优于 baseline 5%,概率极低
- 严重度:高
- 判定:疑似数据选择性问题,需作者提供原始数据

## 疑似信号 2(来自 peer-review):图片可疑
- 核查方法:ELA 图片分析
- 结果:图 3 右下角 ELA 异常,疑似后期编辑
- 严重度:高
- 判定:疑似图片篡改,需作者提供原始图片
```

## 输出格式
学术不端检测报告,含:
- 疑似项列表(来源 / 描述)
- 每条:核查方法 + 核查结果 + 严重度(高/中/低)+ 判定
- 区分无意错误 vs 故意不端的判断
- 后续建议(需作者提供原始材料 / 上报人工核实)

## 约束
- **接收质检员的疑似信号做深入核查**:不自行全文扫描,基于已有信号深入。
- **标注"疑似"不单独判定造假**:学术不端判定需人工委员会,本技能只标"疑似"。
- 不得公开指控作者造假,报告只供内部核实。
- 区分无意错误和故意不端,不混为一谈。
