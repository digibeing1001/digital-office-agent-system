# risk-scoring — 风险评分

## 用途
对科研流水线中的非关键产出(文献综述、实验日志、采集报告、初稿等)做风险评分,判断是否需要暂停上报。与六维评分卡(用于关键产出)不同,本技能用轻量四维模型快速过一遍,放行大多数低风险产出,只在超阈值时升级处理。

适用于:科研秘书对中间产出的快速风控,不替代关键产出的正式评审。

## 触发条件
- 文献研究员提交综述/分类法/引文报告
- 实验员提交实验日志/采集报告
- 写作员提交初稿(非最终稿)
- 任何标注为"非关键产出"的中间件提交

## 操作步骤

### 步骤 1:识别产出类型
判断提交物属于哪一类:
- 综述类(文献综述 / 分类法 / 研究空白报告)
- 数据类(实验日志 / 采集报告 / 元数据)
- 写作类(初稿 / 图注 / 引用整合)
- 流程类(PRISMA 流程 / 可复现清单)

### 步骤 2:按四维风险打分
每个维度 0-25 分,总分 0-100,分越高风险越大:
- **完整性**(0-25):交付物是否覆盖了应有字段/章节/条目
- **准确性**(0-25):内容是否与原始数据/文献一致,有无明显错误
- **及时性**(0-25):是否在约定时间内提交,延期程度
- **合规性**(0-25):是否符合该类型的格式/规范/引用要求

### 步骤 3:标记风险点
- 单项超过 15 分:标记该维度为风险点
- 总分超过 60 分:整体标记为高风险,暂停上报
- 总分 40-60 分:标记为中风险,放行但附风险点
- 总分低于 40 分:低风险,直接放行

### 步骤 4:输出风险报告
按输出格式生成报告,送交科研秘书决策。

## 调用示例
```
输入:文献研究员提交了一份综述,缺少 Related Work 对比,引用格式不一致
评分:
  完整性: 18 (缺少 Related Work 章节)
  准确性: 8 (内容与原文一致)
  及时性: 5 (按时提交)
  合规性: 20 (引用格式多处不一致)
总分: 51 → 中风险,放行但附 2 个风险点
```

## 输出格式
风险评分报告(写入 `outputs/risk/<artifact>-<date>.md`):
- **产出类型**:综述 / 数据 / 写作 / 流程
- **风险分**:0-100
- **维度得分**:完整性 / 准确性 / 及时性 / 合规性 各项分数
- **风险等级**:低 / 中 / 高
- **风险点清单**:每条注明维度 + 具体问题
- **处置建议**:放行 / 放行并附风险点 / 暂停上报并要求返工

## 约束
- 风险分超阈值(60)才暂停上报,否则一律放行,不要过度拦截
- 本技能不用于关键产出(论文终稿/实验方案/申报书),关键产出走六维评分卡
- 四个维度固定,不得自行增删维度
- 评分必须给出具体理由,不得只给数字
- 风险点要可执行(指明问题在哪、怎么修),不要空泛评价
