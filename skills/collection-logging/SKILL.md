# collection-logging — 采集日志

## 用途
记录文献采集过程,包括采集时间、信源、采集数量、去重数量、入库数量和异常,使每次采集可追溯,便于排查采集问题。

适用于:文献采集过程记录、采集量统计、采集异常追踪等场景。

## 触发条件
- 用户要求"记录采集日志""采集日志""记录采集过程"时。
- 每次文献采集完成后需要记录时。
- 用户提到"采集日志""collection log""采集统计"时。

## 操作步骤
1. **记录采集时间**:记录采集开始和结束时间戳。
2. **记录信源**:记录本次采集涉及的信源(RSS / arXiv / S2 / OpenAlex)及每个信源的查询条件。
3. **记录采集数量**:记录每个信源采集到的原始文献数量。
4. **记录去重数量**:记录去重后剔除的重复文献数量。
5. **记录入库数量**:记录最终入库待处理的文献数量。
6. **记录异常**:记录采集过程中的异常(信源不可达 / API 限流 / 解析失败),便于排查。

## 调用示例
```python
import json
from datetime import datetime

log = {
    "采集开始": datetime.now().isoformat(),
    "信源": {
        "arXiv(cs.AI)": {"查询": "cat:cs.AI", "采集数": 50},
        "RSS(Nature AI)": {"采集数": 12},
    },
    "总采集数": 62,
    "去重剔除": 8,
    "入库数": 54,
    "异常": ["RSS(Nature AI) 解析超时,已跳过"],
    "采集结束": datetime.now().isoformat(),
}
with open("logs/collection_20260701.json", "w", encoding="utf-8") as f:
    json.dump(log, f, ensure_ascii=False, indent=2)
```

## 输出格式
采集日志,含:
- 采集时间(开始 / 结束)
- 信源清单(信源名 / 查询条件 / 采集数量)
- 总采集数 / 去重剔除数 / 入库数
- 异常清单(异常类型 / 信源 / 处理方式)

## 约束
- **每次采集必须有日志**:不得无日志采集。
- 异常必须如实记录,不得隐瞒。
- 日志需持久化保存,便于历史追溯。
- 日志格式统一,便于聚合统计。
