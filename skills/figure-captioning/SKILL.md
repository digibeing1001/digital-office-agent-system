# figure-captioning — 图注规范

## 用途
为论文/报告中的图表撰写规范的图注(caption),使图表独立可读——读者不看正文也能理解图表表达什么、数据从哪来、关键发现是什么。

适用于:任何带图表的科研产出(论文/报告/申报书/白皮书)。

## 触发条件
- LaTeX 排版插入图表后
- 论文初稿完成,需要补/改图注
- 用户要求"写图注""改 caption"
- risk-scoring 标记图注不达标

## 操作步骤

### 步骤 1:看图表内容
1. 读取图表本体(图/表)
2. 识别:
   - 图表类型(折线图/柱状图/散点图/表格/示意图/流程图)
   - 横轴/纵轴含义(图)或行列含义(表)
   - 数据来源(实验数据/调研/模拟)
   - 关键观察(最高点/最低点/趋势/异常)
3. 检查图表里的符号/缩写是否在正文定义过

### 步骤 2:写图注
图注按以下结构(可裁剪):

**第 1 句:What(图表在说什么)**
- 用一句话说明图表的主语和动作
- 示例:"不同微调方法在 LongBench 各子任务上的性能对比。"

**第 2 句:Key finding(关键发现)**
- 指出最重要的观察
- 示例:"LoRA(r=8)在大多数子任务上接近全参微调,但在总结任务上落后 3.2 分。"

**第 3 句(可选):How to read(如何解读)**
- 复杂图表需要解读提示
- 示例:"柱状图越高越好,误差线为 3 次运行的标准差。"

**第 4 句(可选):Data source(数据来源)**
- 若数据来自外部,需标注
- 示例:"数据来自 LongBench v2,评估协议见 Section 4.2。"

### 步骤 3:检查独立可读性
独立可读性测试:
1. 把图注和图表放在一起,遮住正文
2. 能否回答以下问题:
   - 图表在展示什么?(What)
   - 最重要的发现是什么?(Key finding)
   - 数据从哪来?(Data source)
   - 符号/缩写什么意思?(Definition)
3. 任一问题答不上,补充图注
4. 符号/缩写必须在图注里定义,不能"见正文 Section X"

### 步骤 4:格式规范
1. 图注位置:图在图下方,表在表上方
2. 图注前缀:
   - 图:"Figure 1: " 或 "图 1:"
   - 表:"Table 1: " 或 "表 1:"
3. 图注长度:1-4 句,不超过 6 行
4. 图注字号:通常比正文小一号(\footnotesize 或 \small)
5. 数字与正文一致(常见错误:图注写 65.2,正文写 65.3)

## 调用示例
```
图表:柱状图,横轴=4种微调方法,纵轴=LongBench平均分,4组柱子对应4个任务
图注(初稿):
  Figure 3: Comparison of fine-tuning methods.
问题:太简略,无 key finding,缩写未定义
图注(修订):
  Figure 3: 不同微调方法在 LongBench 四个子任务上的性能对比。
  LoRA(r=8)在摘要、检索、代码任务上接近全参微调(差距<2分),
  但在总结任务上落后 3.2 分。误差线为 3 次运行的标准差。
  FT=全参微调,PT=Prefix Tuning。
独立可读性:通过(What/Key finding/缩写定义齐全)
```

## 输出格式
图注文本(随 LaTeX 源文件输出):
- 每个图表对应一条图注
- 标注图表编号(Figure 1 / Table 1)
- 图注文本(2-4 句)
- 独立可读性检查结果(通过/未通过 + 缺失项)

## 约束
- 图注必须独立可读,不看正文也能懂
- 符号/缩写必须在图注里定义,不能"见正文"
- 数字必须与正文一致,不得出现 65.2 vs 65.3 这类矛盾
- 图注不写"如图所示"这种废话(读者就在看图)
- 图注不抄正文句子,要写图表特有的观察
- 长度 1-4 句,不超过 6 行,过长说明该写进正文了
- key finding 必须有,不能只描述图表不指出发现
