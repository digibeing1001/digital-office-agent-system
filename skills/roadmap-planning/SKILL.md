# roadmap-planning — 路线图规划

## 用途
基于子问题、假设、创新点,制定研究里程碑、依赖关系、资源需求和时间线,产出可执行的研究路线图。PI 只规划到里程碑级别,不替方法学专家定方案细节。

适用于:研究项目正式启动前,或在阶段切换时重新规划。

## 触发条件
- 子问题和假设清单产出后
- 创新点定位完成
- 用户要求"排个计划""定个时间线"
- 项目阶段切换需要重新规划

## 操作步骤

### 步骤 1:定里程碑
1. 从子问题优先级推导里程碑:P0 子问题对应核心里程碑,P1 对应扩展里程碑
2. 标准里程碑模板(可裁剪):
   - M1:文献调研完成(综述 + 分类法 + 研究空白)
   - M2:研究方案定稿(假设 + 方法 + 指标)
   - M3:数据准备完成(采集 / 清洗 / 划分)
   - M4:实验跑通(baseline + 主实验)
   - M5:分析完成(ablation + 统计检验)
   - M6:论文/报告初稿
   - M7:投稿/提交
3. 每个里程碑标注:交付物、判断人、成功标准

### 步骤 2:排依赖
1. 画依赖图:哪些里程碑必须先完成才能开始下一个
2. 识别可并行的里程碑(如 M1 文献调研 和 M3 数据准备 可并行)
3. 识别关键路径(最长依赖链),关键路径决定总工期

### 步骤 3:估资源
每个里程碑标注所需资源:
- 人力:哪个角色参与(文献研究员 / 方法学专家 / 实验员 / 写作员)
- 算力:GPU 小时数(粗估)
- 数据:需要哪些数据集 / 数据源
- 工具:需要哪些技能/工具支持
- 预算:涉及采购的标注金额(走审批路由)

### 步骤 4:标风险点
每个里程碑标注潜在风险:
- 技术风险:方法可能不 work
- 数据风险:数据可能拿不到 / 质量差
- 时间风险:可能延期
- 资源风险:算力/预算可能不够
每条风险标注:概率(高/中/低)、影响(高/中/低)、应对预案

### 步骤 5:输出路线图
整合成时间线视图,按输出格式产出。

## 调用示例
```
项目:LoRA 长文本研究
M1 文献调研 (Day 1-7) → M2 方案定稿 (Day 8-14) → M3 数据准备 (Day 8-21, 与 M2 并行)
→ M4 实验跑通 (Day 22-35) → M5 分析 (Day 36-42) → M6 初稿 (Day 43-56) → M7 投稿 (Day 57)
关键路径:M1 → M2 → M4 → M5 → M6 → M7 (57 天)
风险:M4 实验可能不 work(概率中,影响高,预案:降级为可行性分析)
```

## 输出格式
研究路线图(写入 `outputs/pi/<project>-roadmap.md`):
- **里程碑清单**:每条含
  - 编号(M1/M2/...)
  - 名称
  - 交付物
  - 判断人
  - 成功标准
  - 起止时间
- **依赖关系图**:文本形式表示(M1 → M2, M1 || M3 表示并行)
- **关键路径**:最长依赖链
- **资源需求表**:每个里程碑的人力/算力/数据/工具/预算
- **风险登记册**:每条含 风险描述 / 概率 / 影响 / 应对预案

## 约束
- 不替方法学专家定方案细节(只定"要做实验",不定"用啥统计方法")
- 里程碑必须可验证(有明确的交付物和成功标准)
- 时间估算要留 buffer,乐观估计乘 1.5
- 风险点必须标应对预案,不能只标风险不给方案
- 关键路径必须显式标注,关键路径延期=项目延期
- 涉及硬管控的里程碑(如 M7 投稿)标注需走审批路由
