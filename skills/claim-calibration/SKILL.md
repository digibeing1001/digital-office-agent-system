# claim-calibration — claim 校准

## 用途
检查论文 claim 是否过强,逐条提取 claim 并与实验证据对比,标记过度声称的 claim,建议弱化措辞,确保 claim 不超过证据支持的范围。

适用于:论文投稿前 claim 自检、过度声称排查、措辞校准等场景。

## 触发条件
- 用户要求"校准 claim""检查 claim""claim 太强了吗"时。
- 论文投稿前需要检查声称是否过头时。
- 用户提到"claim""声称""过度"时。

## 操作步骤
1. **提取所有 claim**:从摘要、引言、结论中提取作者的强陈述(如"显著优于""首次实现""在所有场景下")。
2. **逐条评估强度**:把每条 claim 分级为强(普适/绝对)、中(限定范围)、弱(谨慎)。
3. **对比实验证据**:为每条 claim 找对应的实验证据,评估证据强度(强/中/弱/无)。
4. **标记过强 claim**:强 claim + 弱证据 → 标"过度声称";弱 claim + 强证据 → 标"声称不足"。
5. **建议弱化措辞**:对过度声称的 claim,给出更保守的替代措辞。

## 调用示例
```
输入:论文草稿
输出:claim 校准报告
```
```markdown
## Claim 1(摘要):"本方法在所有 benchmark 上显著优于 SOTA"
- 强度:强(普适 + "显著")
- 证据:仅在 3 个 benchmark 上测试,其中 1 个未达显著
- 判定:过度声称
- 建议措辞:"本方法在 3 个 benchmark 中的 2 个上优于 SOTA(p<0.05)"

## Claim 2(结论):"首次将 X 引入 Y 任务"
- 强度:强("首次")
- 证据:未查相关工作
- 判定:需查证
- 建议:先做相关工作核查,确认无前例后再用"首次"。
```

## 输出格式
claim 校准报告,含:
- claim 列表(原文 + 位置)
- 每条 claim 的强度评估
- 对应的实验证据及证据强度
- 判定(过度声称 / 称声称不足 / 合适)
- 建议措辞(对过度声称的 claim)

## 约束
- **claim 不能超过证据支持的范围**:强 claim 必须有强证据。
- "显著优于"必须有统计显著性检验支撑。
- "首次""SOTA"必须查证相关工作后才能用。
- 泛化 claim("所有场景""任意")必须有多场景实验,否则标过度声称。
