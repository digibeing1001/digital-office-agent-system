# authorship-verification — 署名验证

## 用途
验证论文署名是否符合目标期刊/会议的署名规范,检查作者列表、贡献声明、通讯作者、利益冲突声明和作者顺序合理性。

适用于:论文投稿前署名自检、作者贡献核查、署名规范审查等场景。

## 触发条件
- 用户要求"验证署名""检查作者列表""署名规范"时。
- 论文投稿前需要核对署名时。
- 用户提到"署名""authorship""贡献声明"时。

## 操作步骤
1. **检查作者列表**:确认所有作者姓名、单位、邮箱完整,无遗漏或多余。
2. **检查贡献声明**:核对每位作者的贡献是否明确说明(按 CRediT 贡献分类),无"所有作者均贡献"的模糊表述。
3. **检查通讯作者**:确认通讯作者标识正确,联系邮箱有效。
4. **检查利益冲突声明**:确认是否有利益冲突声明(即便"无冲突"也要声明)。
5. **检查顺序合理**:作者顺序是否反映贡献大小,第一作者和最后作者位置是否符合领域惯例。
6. **对齐目标期刊/会议规范**:按目标期刊/会议的署名要求逐项核对。

## 调用示例
```
输入:论文草稿 + 目标期刊/会议署名规范
输出:署名验证报告
```
```markdown
## 检查项
- 作者列表:✅ 5 位作者,单位邮箱齐全
- 贡献声明:⚠️ 仅写"所有作者均参与",需按 CRediT 细化
- 通讯作者:✅ 张三,邮箱有效
- 利益冲突:❌ 缺少声明
- 顺序:✅ 第一作者为主要贡献者

## 修改建议
1. 按 CRediT 分类细化每位作者贡献(如 A:方法设计,B:实验)。
2. 补充利益冲突声明(即便无冲突也要写"作者声明无利益冲突")。
```

## 输出格式
署名验证报告,含:
- 各检查项状态(✅ 通过 / ⚠️ 待改 / ❌ 缺失)
- 具体问题描述
- 修改建议
- 与目标期刊/会议规范的对齐情况

## 约束
- **按期刊/会议署名规范**:不同期刊/会议要求不同,必须按目标规范核对。
- 贡献声明不得用"所有作者均贡献"等模糊表述,必须细化到个人。
- 利益冲突声明必须有(即便"无冲突")。
- 不得随意增删作者,作者变更需所有作者同意。
