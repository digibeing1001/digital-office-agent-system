# 学术写作技能(Academic Writing)

## 用途
按学术体裁规范起草各类科研文本(论文 / 申报书 / 白皮书 / 调研报告),遵循"先骨架后血肉、先方法后引言、最后写摘要"的写作顺序,确保叙事一致、引用有据、图表独立可读,产出可投稿或可提交的高质量草稿。

适用于:期刊/会议论文起草、申报书撰写、技术白皮书、调研报告、学位论文章节等场景。

## 触发条件
满足以下任一条件即激活本技能:
- 用户要求"写论文""起草申报书""写白皮书""写报告""起草 introduction""写 abstract"
- 已有实验结果或调研材料,需要组织成规范的学术文本
- 需要按 NeurIPS/ICML/ICLR/ACL 等会议模板写作
- 需要切换不同体裁的写作风格

## 操作步骤

### 步骤 1:确定体裁和结构
1. 明确写作体裁,不同体裁侧重不同:
   - **论文**:重方法证据,核心是"我做了什么 + 为什么有效 + 怎么证明"
   - **申报书**:重价值创新,核心是"为什么做 + 做了有什么用 + 凭什么我能做成"
   - **白皮书**:重洞察建议,核心是"现状是什么 + 趋势在哪 + 该怎么办"
   - **调研报告**:重调研发现,核心是"查了什么 + 发现什么 + 结论建议"
2. 按体裁套用标准结构(论文:IMRaD+;申报书:背景/目标/内容/方案/基础/计划)
3. 列出每一节的标题和要点,形成骨架大纲
4. 与用户确认骨架后再进入起草

### 步骤 2:写一句话叙事线
1. 用一句话概括整篇文本的核心 claim
   - 论文示例:"我们提出 X 方法,通过 Y 机制,在 Z benchmark 上比 SOTA 提升 N%,且开销更低。"
   - 申报书示例:"针对 X 问题,本项目将研究 Y 方法,实现 Z 目标,支撑 W 应用。"
2. 这句话是全文的"北极星",后续每一段都要服务于它
3. 叙事线写入文档顶部,作为写作时反复对照的锚点

### 步骤 3:按推荐顺序起草
**重要:不按阅读顺序写,按依赖顺序写。**

起草顺序及理由:

1. **Method(方法)**:最先写,因为这是你最确定的内容
   - 写清问题形式化、方法整体架构、各组件细节、训练/推理流程
   - 用数学符号要一致(定义一次,全文沿用)
   - 每个设计选择都要给理由(为什么这样而不那样)
2. **Results(结果)**:基于实验数据写
   - 主结果表 + ablation 表 + 分析性实验
   - 先描述现象,再给解释,最后连接到 claim
   - 数字与实验记录严格一致,不得四舍五入美化
3. **Discussion(讨论)**:解读结果的意义和局限
   - 结果超出/符合/低于预期的原因分析
   - 与相关工作的对比定位
   - 局限性(必须诚实写,审稿人会找)
   - 未来工作方向
4. **Introduction(引言)**:此时已知全文,反向写引言
   - 第 1 段:大背景和领域重要性
   - 第 2 段:现有方法的局限(对应你要解决的问题)
   - 第 3 段:你的方法和核心贡献(对应叙事线)
   - 第 4 段:主要结果亮点(数字)
   - 第 5 段:论文组织结构
5. **Related Work(相关工作)**:基于文献检索技能的产出
   - 按主题分组,不按时间流水账
   - 每组最后说明"与本文的区别"
   - 引用要平衡,不漏关键工作
6. **Conclusion(结论)**:浓缩全文
   - 重申核心贡献(与摘要呼应但不重复措辞)
   - 总结主要结果
   - 局限与展望
7. **Abstract(摘要)**:最后写,150-250 字
   - 背景 1-2 句 → 问题 1 句 → 方法 1-2 句 → 结果 1-2 句(含关键数字)→ 意义 1 句
   - 每个词都要精打细算,摘要决定论文是否被读

### 步骤 4:整合引用
1. 引用必须基于真实文献(由文献检索技能产出),不得编造
2. 暂时找不到合适引用的位置,标注 `[待补引用:说明需要引用什么样的工作]`
3. 引用密度:Introduction 和 Related Work 引用密集,Method 适度,Results 少量(对比时)
4. 自引控制:不超过 20%,避免显得自吹自擂
5. 引用格式遵循目标会议/期刊规范(数字 [1] / 作者-年 (Smith, 2020))

### 步骤 5:写图表说明
1. **图表必须独立可读**:读者不看正文也能理解图表表达什么
2. Caption 结构:
   - 第 1 句:图表在说什么(What)
   - 第 2 句:关键观察(Key finding)
   - 第 3 句(可选):如何解读(Hover/Read 提示)
3. 图表数字必须与正文叙述的数字一致(常见翻车点)
4. 表格设计:重要对比列在前,加粗最优值(标注 * 或粗体),单位写清
5. 图分辨率:矢量图优先(PDF/SVG),位图 ≥ 300 DPI

### 步骤 6:写摘要(最后一步)
1. 长度:150-250 字(按会议要求,NeurIPS 通常 250 字以内)
2. 不引用文献、不引用图表编号
3. 必须包含可量化的关键结果数字
4. 与 Introduction 第 1 段和 Conclusion 第 1 段呼应但不雷同
5. 写完后朗读一遍,检查是否"听起来像论文"

### 步骤 7:自查一致性
草稿完成后,逐项检查:

1. **claim 一致性**:
   - 摘要的 claim ⊆ 引言的 claim ⊆ 结论的 claim
   - 不得摘要说"显著提升"而结论说"略有改善"
2. **数字一致性**:
   - 摘要的数字 = 正文表格的数字 = 图表 caption 的数字
   - 不得不同位置出现不同数字(常见错误)
3. **术语一致性**:
   - 同一概念全文用同一个词(不要 interchangeably 用 method/approach/technique)
4. **符号一致性**:
   - 数学符号定义一次,全文沿用,大小写/上下标不漂移
5. **时态一致性**:
   - 描述本方法用现在时,描述实验过程用过去时,描述事实用现在时
6. **引用完整性**:
   - 所有 [待补引用] 是否已补全或显式标注
   - BibTeX 中每条 entry 是否在正文被引用(未引用的清理)

## 输出格式
产出以下文件(建议放在 `outputs/writing/<project>-<date>/`):
- `narrative-line.md`:一句话叙事线 + 骨架大纲
- `draft.md`:完整草稿(按推荐顺序组织)
- `figures-captions.md`:图表清单 + caption 草稿
- `abstract.md`:摘要定稿
- `consistency-check.md`:一致性自查报告
- `references.bib`:引用文献 BibTeX(由文献检索技能产出)
- `genre-adaptation.md`:体裁适配说明(若切换体裁)

## 约束
- **绝不编造引用**:无依据的引用必须标注 `[待补引用]`,不得伪造
- **每句有依据**:陈述事实必须有来源(实验数据 / 引用 / 定义),否则标 `[待补依据]`
- **不擅自改核心 claim**:用户给的 claim 不得在写作中弱化或夸大
- **不美化数字**:结果数字严格按实验记录,不得四舍五入或挑选有利数字
- **不抄袭**:改写他人观点必须标注引用,直接引用必须加引号
- **不堆砌术语**:专业术语首次出现给定义,后续沿用
- **体裁不混用**:论文不写成申报书语气,申报书不写成论文结构
- **不省略局限**:局限性必须写,审稿人一定会找

## 依赖工具/API
- **文献支撑**:literature-retrieval 技能产出(引用 + 综述 + BibTeX)
- **实验数据**:experiment-design 技能产出(指标 + 表格 + 统计检验)
- **图表生成**:matplotlib / seaborn / plotly / TikZ / draw.io
- **LaTeX 模板**:NeurIPS / ICML / ICLR / ACL 官方模板
- **语法检查**:Grammarly / LanguageTool(可选)
- **引用管理**:Zotero / BibLaTeX / pandoc-citeproc
- **协作版本控制**:Git(追踪草稿演化)

## 体裁切换说明
当用户要求切换体裁时,调整以下要素:

| 要素 | 论文 | 申报书 | 白皮书 | 调研报告 |
|------|------|--------|--------|----------|
| 核心 | 方法证据 | 价值创新 | 洞察建议 | 调研发现 |
| 语气 | 客观严谨 | 价值导向 | 权威前瞻 | 中立全面 |
| 引用 | 密集 | 适度 | 适度 | 密集 |
| 数据 | 实验数据 | 预期效益 | 行业数据 | 调研数据 |
| 结构 | IMRaD+ | 背景/目标/内容/方案/基础/计划 | 现状/趋势/建议 | 概述/方法/发现/建议 |
| 长度 | 8-10 页 | 30-50 页 | 10-30 页 | 20-100 页 |
