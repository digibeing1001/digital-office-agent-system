# taxonomy-tagging — 分类标签

## 用途
对入库文献打分类标签,对齐文献研究员建好的分类法,使文献可按主题检索和聚合;标签冲突时标"待回填"。

适用于:文献入库分类、主题标签打标、分类法对齐等场景。

## 触发条件
- 用户要求"打分类标签""给文献分类""打标签"时。
- 新文献入库需要打分类时。
- 用户提到"taxonomy""分类标签""文献分类"时。

## 操作步骤
1. **读文献元数据**:读取文献的标题、摘要、关键词、来源等元数据。
2. **对齐文献研究员建的分类法**:加载文献研究员维护的分类法(主题树 / 标签集),不自行新建分类。
3. **打标签**:按元数据匹配分类法中的标签,可打多个标签(主主题 + 次主题)。
4. **冲突时标"待回填"**:当文献难以明确归类或匹配多个冲突标签时,标"待回填",交文献研究员人工处理。

## 调用示例
```python
# 分类法示例(由文献研究员维护)
taxonomy = {
    "机器学习": ["深度学习", "强化学习", "联邦学习"],
    "自然语言处理": ["大语言模型", "机器翻译", "问答系统"],
}

# 给文献打标签
def tag_paper(title, abstract, taxonomy):
    tags = []
    for category, subcats in taxonomy.items():
        for sub in subcats:
            if sub in title or sub in abstract:
                tags.append(f"{category}/{sub}")
    return tags if tags else ["待回填"]

tags = tag_paper("Scaling Laws for Large Language Models", "...")
print(tags)  # ['自然语言处理/大语言模型']
```

## 输出格式
带分类标签的文献记录,含:
- 文献元数据(标题 / 摘要 / 来源)
- 分类标签(主主题 / 次主题,可多个)
- 标签状态(已分类 / 待回填)

## 约束
- **对齐文献研究员的分类法,不自行建分类法**:分类法由文献研究员维护,本技能只做匹配。
- **空库时按 arXiv 粗分类**:分类法尚未建立时,可暂用 arXiv 分类(cs.AI / cs.CL 等)做粗分类,待分类法建好后回填。
- 标签冲突或不确定时标"待回填",不强行归类。
- 标签变更需记录,便于追溯。
