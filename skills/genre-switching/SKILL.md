# genre-switching — 体裁切换

## 用途
按产出类型(论文/报告/申报书/白皮书)切换写作体裁,加载对应模板,按体裁特有的结构和语气组织内容,避免体裁混用(论文写成申报书语气、申报书写成论文结构)。

适用于:同一研究内容需要输出不同体裁时,或用户指定了产出体裁。

## 触发条件
- 用户指定产出体裁("写论文""写申报书""写白皮书""写报告")
- 同一研究内容需要从一种体裁转成另一种
- 写作员开始新文档前

## 操作步骤

### 步骤 1:识别体裁
明确产出属于哪一类:

| 体裁 | 核心诉求 | 典型场景 |
|------|----------|----------|
| 论文 | 方法证据 | 期刊/会议投稿 |
| 申报书 | 价值创新 | 基金/项目申报 |
| 白皮书 | 洞察建议 | 行业/技术发布 |
| 调研报告 | 调研发现 | 内部决策支持 |
| 学位论文 | 系统训练 | 毕业 |

### 步骤 2:加载对应模板
按体裁加载结构模板:

**论文(IMRaD+)**:
- Abstract / Introduction / Related Work / Method / Experiments / Discussion / Conclusion

**申报书**:
- 背景/立项依据 / 研究目标 / 研究内容 / 研究方案 / 研究基础 / 进度计划 / 预期成果

**白皮书**:
- 摘要 / 现状分析 / 趋势研判 / 关键技术 / 应用场景 / 发展建议

**调研报告**:
- 概述 / 调研方法 / 主要发现 / 分析讨论 / 结论建议

**学位论文**:
- 绪论 / 相关工作 / 核心方法(多章) / 实验 / 总结展望

### 步骤 3:按体裁结构组织内容
1. 把已有研究内容(子问题/假设/方法/结果)映射到体裁结构
2. 不同体裁的内容侧重不同:
   - 论文:重方法细节和实验证据,Method 和 Experiments 是核心
   - 申报书:重价值和可行性,研究目标和方案是核心
   - 白皮书:重洞察和趋势,分析章节是核心
   - 调研报告:重发现和建议,主要发现是核心
3. 同一份实验数据,在不同体裁里呈现方式不同:
   - 论文:完整表格 + 统计检验
   - 申报书:关键数字 + 预期效益
   - 白皮书:趋势图 + 行业对比
   - 报告:摘要表 + 结论建议

### 步骤 4:检查语气和规范
1. **论文语气**:客观严谨,第三人称,被动语态多,避免主观判断
2. **申报书语气**:价值导向,既要表达信心又要务实,避免过度承诺
3. **白皮书语气**:权威前瞻,有立场但不偏激,适合决策者阅读
4. **报告语气**:中立全面,呈现事实不下定论(除非有充分证据)
5. 检查长度:
   - 论文:8-10 页(会议) / 20-40 页(期刊)
   - 申报书:30-50 页
   - 白皮书:10-30 页
   - 报告:20-100 页

### 步骤 5:体裁一致性检查
1. 全文语气一致,不前后切换
2. 结构符合体裁规范(不把论文写成 IMRaD 又加申报书的"研究基础")
3. 引用密度符合体裁(论文密集,白皮书适度,申报书适度)
4. 数字呈现方式一致(全文用同一种表格风格)

## 调用示例
```
输入:已有研究内容(LoRA 长文本研究的方法/实验/结果)
任务:同时产出论文和申报书
论文版:
  结构:Abstract / Intro / Related Work / Method / Experiments / Conclusion
  语气:客观严谨
  数据:完整实验表 + 统计检验
  长度:9 页(NeurIPS 模板)
申报书版:
  结构:背景 / 目标 / 内容 / 方案 / 基础 / 计划
  语气:价值导向
  数据:关键数字 + 预期效益
  长度:35 页
```

## 输出格式
对应体裁的文档(写入 `outputs/writing/<project>-<genre>/`):
- **体裁标识**:论文 / 申报书 / 白皮书 / 报告
- **结构模板**:章节大纲
- **内容映射表**:研究内容 → 体裁章节的映射
- **正文**:按体裁结构组织
- **体裁一致性检查**:语气/结构/引用/长度的检查结果

## 约束
- 体裁不同结构不同,不混用(论文不写成申报书结构)
- 语气必须与体裁匹配,论文不写申报书的"本项目将实现..."
- 同一内容转体裁时,数据呈现方式要调整,不是简单复制
- 用户未指定体裁时必须问清,不得擅自假设
- 切换体裁时保留研究内容的事实(方法/数据/结果),只改组织和呈现
- 长度遵循体裁规范,不超不缺
