# plagiarism-check — 查重

## 用途
对文本和代码同时做查重,文本用 MinHash 查重、代码用 JPlag 查重,生成查重报告,重点看核心内容(方法/数据/结论)是否实质性重复,而非只看相似度比例。

适用于:论文投稿前查重、代码原创性检查、学术不端排查等场景。

## 触发条件
- 用户要求"查重""plagiarism""查抄袭"时。
- 论文投稿前需要做查重时。
- 用户提到"文本查重""代码查重""相似度"时。

## 操作步骤
1. **文本用 MinHash 查重**:对论文文本做分词和 shingle,用 MinHash+LSH 与知识库已入库文献比对,输出相似度排序。
2. **代码用 JPlag 查重**:对实验代码用 JPlag 做结构化比对,检测代码块级抄袭(不只是文本相似)。
3. **生成查重报告**:汇总文本和代码查重结果,列出高相似度匹配项及位置。
4. **重点看核心内容重复**:对高相似度项人工复核,判断是否核心内容(方法/数据/结论/关键算法)实质性重复,套话和通用表述重复不算。

## 输出格式
查重报告,含:
- 文本相似度:Top-K 相似文献列表(Jaccard 相似度 + 重复段落定位)
- 代码相似度:Top-K 相似代码块(相似度 + 代码片段定位)
- 核心内容重复标注(哪些是实质性重复,哪些是通用表述)
- 复核结论(疑似抄袭 / 引用不规范 / 无问题)

## 约束
- **查重看实质不看比例**:核心内容(方法/数据/结论)重复才是问题,套话/通用表述重复不算。
- 文本和代码必须分开查,用各自适合的方法。
- 高相似度结果需人工复核,MinHash/JPlag 是近似算法。
- 查重知识库需持续更新,否则漏报。
- 不单独判定抄袭,疑似项标"需人工核实"。
