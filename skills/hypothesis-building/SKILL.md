# hypothesis-building — 假设构建

## 用途
基于 PI 拆解出的子问题,提出可证伪的科学假设。每个假设要明确预测、前提条件、验证方式,确保后续实验设计有明确的靶子可打。

适用于:子问题清单产出后、实验方案设计前。

## 触发条件
- research-question-structuring 产出子问题清单
- 用户要求针对某子问题提假设
- 实验设计阶段需要明确的假设作为靶子

## 操作步骤

### 步骤 1:遍历子问题
1. 读取子问题清单,按优先级逐个处理(P0 先做)
2. 跳过纯描述性问题(描述性问题不强制要假设)
3. 比较性/关联性/因果性/预测性问题必须提假设

### 步骤 2:针对每个子问题提假设
每个假设包含四个字段:
- **假设陈述(H)**:用一句话陈述预测的关系,如"X 越大,Y 越小"
- **预测(P)**:可观测的具体预测,如"在 Z 任务上,LoRA(r=8)的准确率比全参低不超过 2%"
- **前提条件(C)**:假设成立的前置条件,如"在模型规模 ≥ 7B 时"
- **验证方式(V)**:用什么实验/数据来证伪,如"在 LongBench 上对比 LoRA vs 全参"

### 步骤 3:检查可证伪性
逐条用 Popper 可证伪性检查:
- 假设能否被某个观测结果证伪?不能证伪的(如"X 可能影响 Y")必须重写
- 预测是否具体到可量化?笼统的"会有提升"必须改成数字或方向
- 验证方式是否可执行?需要不可得的数据/设备的假设需要降级或更换

### 步骤 4:标注假设类型
- **零假设(H0)**:无差异/无关联(用于统计检验)
- **备择假设(H1)**:有差异/有关联(研究真正想验证的)
- **方向性假设**:明确方向(大于/小于)
- **非方向性假设**:只说不同,不定方向

### 步骤 5:标注假设间关系
- 互斥假设:不能同时成立
- 递进假设:H2 依赖 H1 成立
- 独立假设:互不影响

## 调用示例
```
子问题:LoRA 与全参微调在长文本任务上的性能差异有多大?
假设 H1:
  H: LoRA 微调在长文本任务上性能低于全参微调
  P: 在 LongBench 上,LoRA(r=8)平均分比全参低 2-5 分
  C: 模型规模 7B-13B,训练数据相同
  V: LongBench 多任务对比,相同随机种子跑 3 次取均值
类型:方向性备择假设
对应零假设 H0: LoRA 与全参在 LongBench 上无显著差异
```

## 输出格式
假设清单(写入 `outputs/pi/<project>-hypotheses.md`):
- **对应子问题编号**:Q1 / Q2 / ...
- **假设编号**:H1 / H2 / ...
- **假设陈述(H)**
- **预测(P)**
- **前提条件(C)**
- **验证方式(V)**
- **假设类型**:零/备择/方向性/非方向性
- **可证伪性检查**:通过/未通过(附原因)
- **假设间关系**:互斥/递进/独立 + 关联假设编号

## 约束
- 假设必须可证伪,不可证伪的重写或删除
- 预测必须具体到可量化(数字或方向),禁止"会有提升""可能更好"这类表述
- 不替方法学定具体统计方法,只定"预测什么"
- 前提条件必须显式列出,不得默认隐含
- 描述性问题不强制提假设,但比较/关联/因果/预测类必须有假设
- 零假设和备择假设成对出现,缺一不可
