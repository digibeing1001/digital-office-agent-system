# citation-verification — 引文验证

## 用途
对论文/报告中的引用做三重交叉验证,通过 Semantic Scholar、OpenAlex、Crossref 三个独立源核对引用的存在性和元数据一致性,标记未验证或不一致的引用,确保不出现编造引用或元数据错误。

适用于:论文投稿前、申报书提交前、综述定稿前的引用核查。

## 触发条件
- 论文/报告初稿完成,引用已整合
- 用户要求"核查引用""验证引文"
- risk-scoring 标记引用合规性风险
- 投稿/提交前(硬管控触发前的必经步骤)

## 操作步骤

### 步骤 1:提取引用
1. 从正文解析所有引用标记(如 [1] / (Smith, 2020) / \cite{key})
2. 从参考文献列表提取每条引用的元数据:标题 / 作者 / 年份 / 来源(期刊/会议)/ DOI / URL
3. 建立引用标记 ↔ 参考文献条目的映射
4. 检查正文标记和参考文献列表是否一一对应(孤儿标记、孤儿条目)

### 步骤 2:三源查询
对每条引用,分别向三个源查询:
1. **Semantic Scholar**:按标题/DOI 查询,获取论文 ID 和元数据
2. **OpenAlex**:按标题/DOI 查询,获取 Work ID 和元数据
3. **Crossref**:按 DOI 查询,获取元数据(以 DOI 为权威)

查询策略:
- 优先用 DOI 查(最准)
- 无 DOI 用标题 + 第一作者查
- 标题查不到用模糊匹配(编辑距离 ≤ 3)

### 步骤 3:交叉验证
对每条引用,检查:
1. **存在性**:三个源里至少一个能查到
   - 三源都查到:存在性确认
   - 部分源查到:存在性确认,记录哪些源没查到
   - 三源都查不到:标记"未验证"
2. **元数据一致性**:核对标题/作者/年份/来源是否与正文引用一致
   - 标题:允许大小写和标点差异,实质内容必须一致
   - 作者:姓名顺序和数量必须一致
   - 年份:必须完全一致
   - 来源:期刊/会议名必须一致(允许缩写差异)

### 步骤 4:标记不一致
对每条引用标注验证状态:
- **已验证(一致)**:存在且元数据一致
- **已验证(不一致)**:存在但元数据有出入,标注具体差异
- **未验证**:三源都查不到,可能是编造或元数据错误严重
- **部分验证**:只有部分源查到,标注哪些源

### 步骤 5:生成验证报告
按输出格式生成报告,送交写作员修正。

## 调用示例
```
引用 [3]:Smith et al., 2023, "LoRA: Low-Rank Adaptation", NeurIPS
查询:
  Semantic Scholar: 找到,paper_id=xxx,标题一致,作者一致,年份一致
  OpenAlex: 找到,work_id=xxx,标题一致
  Crossref: 找到,DOI=10.48550/arXiv.2106.09685
状态:已验证(一致)
---
引用 [7]:Wang 2024, "Survey of PEFT", arXiv
查询:
  Semantic Scholar: 未找到
  OpenAlex: 未找到
  Crossref: 未找到
状态:未验证 → 标记可疑,要求写作员核实
```

## 输出格式
引文验证报告(写入 `outputs/lit/<project>-citation-verify.md`):
- **引用总数 / 已验证 / 不一致 / 未验证 / 部分验证** 统计
- **逐条验证表**:每条含
  - 引用标记(如 [3])
  - 正文元数据(标题/作者/年份/来源)
  - 三源查询结果(各源是否查到 + 返回的元数据)
  - 验证状态(已验证一致/已验证不一致/未验证/部分验证)
  - 不一致点(如有)
- **未验证引用清单**:重点标注,要求写作员核实
- **孤儿标记/孤儿条目清单**

## 约束
- 三源都查不到的引用标记为"未验证",不得默认为真实
- 元数据不一致必须标注具体差异,不得笼统说"有问题"
- 不得只查一个源就下结论,必须三源交叉
- DOI 是最权威的标识,有 DOI 优先用 DOI 查
- 模糊匹配需谨慎,编辑距离阈值 ≤ 3,超过不算匹配
- 未验证的引用不得放行到投稿/提交环节
