# outlier-detection — 异常值检测

## 用途
检测实验数据中的异常值,用统计方法识别并标记,调查异常原因,决定保留或剔除,并记录理由,防止异常值污染结论或误剔真实信号。

适用于:实验数据清洗、异常结果排查、数据质量审查、离群样本分析等场景。

## 触发条件
- 用户要求"检测异常值""找离群点""清洗数据"时。
- 实验结果出现可疑极端值时。
- 用户提到"outlier""离群""异常值"时。

## 操作步骤
1. **画分布图**:画箱线图 / 直方图 / 散点图,目视检查是否有明显的离群点。
2. **用统计方法检测**:用 IQR 法(超出 1.5×IQR)或 Z-score 法(|z|>3)做定量检测,记录检测方法和阈值。
3. **标记异常**:把检测到的异常值逐个标记,记录样本 ID、数值、检测方法。
4. **调查原因**:对每个异常值调查成因(测量错误 / 数据损坏 / 真实极端样本 / 实验故障),区分"真异常"和"真信号"。
5. **决定保留或剔除**:按原因决定,剔除必须记录理由,保留也说明为什么保留。

## 调用示例
```python
import numpy as np

# IQR 法
q1, q3 = np.percentile(data, [25, 75])
iqr = q3 - q1
lower, upper = q1 - 1.5 * iqr, q3 + 1.5 * iqr
outliers = [x for x in data if x < lower or x > upper]

# Z-score 法
z = (np.array(data) - np.mean(data)) / np.std(data)
outlier_idx = np.where(np.abs(z) > 3)[0]
```

## 输出格式
异常值报告,含:
- 检测方法(IQR / Z-score)及阈值
- 异常值列表(样本 ID / 数值 / 检测方法)
- 每个异常的原因调查
- 处理决定(保留 / 剔除)及理由

## 约束
- **剔除异常值必须记录理由**:不得无理由剔除,不得为迎合结论选择性剔除。
- 不得仅凭"看着不顺眼"就剔除,必须有统计依据或明确原因。
- 真实极端样本(如长尾分布的自然尾点)不得当异常剔除。
- 剔除后需评估对结论的影响,敏感性分析。
