# prisma-screening — PRISMA 筛选

## 用途
按 PRISMA(Preferred Reporting Items for Systematic Reviews and Meta-Analyses)标准对检索到的文献进行四阶段筛选:识别 → 筛选 → 纳入 → 排除,记录每个排除决定的具体理由,生成 PRISMA 流程图数据和纳入/排除文献清单。

适用于:系统综述、Meta 分析、文献综述的文献筛选环节。

## 触发条件
- 文献检索完成,有一批待筛选文献
- 用户要求"按 PRISMA 标准筛选"
- 系统综述/Meta 分析项目的筛选阶段

## 操作步骤

### 步骤 1:识别(Identification)
1. 汇总所有数据库检索结果(Semantic Scholar / OpenAlex / PubMed / arXiv 等)
2. 记录每条来源的初始命中数
3. 合并去重,记录去重前后数量
4. 输出:总识别数 = 各数据库命中数之和 - 重复数

### 步骤 2:标题摘要筛选(Screening)
1. 制定筛选标准(纳入标准 + 排除标准),与 PI 确认
2. 逐篇看标题和摘要,判定:
   - 纳入:明显符合,进入全文筛选
   - 排除:明显不符合,记录排除理由
   - 不确定:进入全文筛选(不轻易排除)
3. 记录每篇排除文献的排除理由(从预设理由库选)
4. 输出:标题摘要筛选后剩余数

### 步骤 3:全文筛选(Eligibility)
1. 获取标题摘要筛选通过文献的全文
2. 逐篇读全文,按纳入/排除标准判定:
   - 纳入:符合所有纳入标准,不触犯任何排除标准
   - 排除:记录具体排除理由
3. 全文获取不到的,标注"全文不可得"并单独统计
4. 输出:全文筛选后纳入数

### 步骤 4:记录排除理由
每个排除决定必须记录:
- 文献标识(DOI / 标题 / 作者-年份)
- 排除阶段(标题摘要 / 全文)
- 排除理由(从理由库选,理由库预设如下)
  - 不在研究范围内
  - 非目标研究类型(如只要 RCT,排掉观察性)
  - 非目标人群/对象
  - 非目标干预/方法
  - 数据不全无法提取
  - 质量不达标
  - 重复发表
  - 全文不可得
  - 非 Peer Review(视情况)

### 步骤 5:生成 PRISMA 流程图数据
按 PRISMA 2020 标准生成流程图各节点数据:
- 识别:各数据库命中数 + 去重数
- 筛选:标题摘要筛选排除数 + 理由分布
- 纳入:全文筛选排除数 + 理由分布
- 最终纳入数

## 调用示例
```
输入:Semantic Scholar 命中 234 篇,OpenAlex 命中 187 篇
去重:重复 89 篇,识别总数 = 234 + 187 - 89 = 332
标题摘要筛选:排除 245 篇(理由:不在范围 180,非目标类型 65)
全文筛选:获取全文 87 篇,排除 32 篇(理由:数据不全 18,质量不达标 14)
全文不可得:3 篇
最终纳入:87 - 32 = 55 篇
PRISMA 流程图:332 → 87 → 55
```

## 输出格式
- **PRISMA 流程图数据**(写入 `outputs/lit/<project>-prisma.md`):
  - 各数据库命中数
  - 去重数
  - 标题摘要筛选排除数 + 理由分布
  - 全文筛选排除数 + 理由分布
  - 全文不可得数
  - 最终纳入数
- **纳入文献清单**:每条含 DOI / 标题 / 作者 / 年份 / 来源
- **排除文献清单**:每条含 DOI / 标题 / 排除阶段 / 排除理由

## 约束
- 每个排除决定必须记录理由,理由从理由库选,不得"感觉不对"就排除
- 不确定的文献不轻易排除,送全文筛选
- 纳入/排除标准必须在筛选前与 PI 确认,筛选中不得随意改
- 全文获取不到的单独统计,不算作排除
- 流程图数据必须可追溯(各节点数量能对得上)
- 筛选过程建议双盲(两人独立筛,分歧讨论),单人筛选需标注
