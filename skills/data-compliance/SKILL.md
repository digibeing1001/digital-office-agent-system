# data-compliance — 数据合规

## 用途
检查论文实验数据使用是否合规,识别数据类型,检查数据来源、授权、隐私处理(匿名化)和数据使用声明,确保涉及个人数据时有匿名化和授权。

适用于:论文数据合规审查、隐私数据检查、数据使用声明核查等场景。

## 触发条件
- 用户要求"检查数据合规""数据合规审查""隐私检查"时。
- 论文使用了可能涉及隐私的数据时。
- 用户提到"数据合规""data compliance""匿名化"时。

## 操作步骤
1. **识别数据类型**:判断所用数据是公开数据集 / 自建数据集 / 个人数据 / 敏感数据,标注类型。
2. **检查数据来源**:每个数据集的来源是否明确,是否为合法渠道获取,是否有引用。
3. **检查授权**:数据集的使用许可(如 CC-BY / MIT / 商业授权 / 受限使用)是否覆盖论文用途。
4. **检查隐私处理(匿名化)**:涉及个人数据时,是否做了匿名化 / 去标识化,匿名化是否充分(无法反推身份)。
5. **检查数据使用声明**:论文是否声明数据来源、许可和(若涉及)隐私处理方式。

## 调用示例
```
输入:论文草稿 + 数据集列表(名称/来源/许可)
输出:数据合规报告
```
```markdown
## 数据集清单
1. 数据集 A(公开)
   - 来源:HuggingFace,许可 CC-BY 4.0
   - 授权:✅ 覆盖研究用途
   - 隐私:不涉及个人数据
2. 数据集 B(自建,含用户文本)
   - 来源:自采集
   - 授权:需确认采集时用户是否知情同意
   - 隐私:⚠️ 未说明匿名化,需补充
```

## 输出格式
数据合规报告,含:
- 数据集清单(名称 / 类型 / 来源 / 许可)
- 各数据集合规状态(✅ 合规 / ⚠️ 待改 / ❌ 不合规)
- 隐私处理检查(是否涉及个人数据 / 匿名化是否充分)
- 修改建议

## 约束
- **涉及个人数据必须有匿名化和授权**:缺一不可,否则标"不合规"。
- 不得使用来源不明的数据。
- 数据许可必须覆盖论文实际用途,商用数据集用于学术研究也需确认许可。
- 匿名化必须充分,K-匿名或其他标准需说明。
