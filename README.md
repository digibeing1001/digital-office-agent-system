# Digital Office · 写作团队版

> 一支专门帮你写新媒体文章的 AI 团队，8 个角色各管一摊，7 道关卡道道把关。

## 当前完成度

写作团队版已具备 GUI readiness 的后端闭环：`gui-state` 提供首页状态快照，`settings-update` 支持安装后的偏好增量更新，首次启动会生成 default secretary persona，并把写作工作流、任务、审批、通知、数字员工、技能安装、循环状态和上下文合约一起暴露给前端。用户在界面里同意继续后，后端会返回可执行的 `resume_signal`，由 GUI 调用下一步命令，而不是让 AI 再解释一遍准备怎么做。

## 给谁用

只要你做新媒体内容创作、又被 AI 写作的「AI 味」「没人味」「方向跑偏」折磨过，这支团队就适合你：

- **公众号作者**：一个人写不过来，又不想找人代写因为代写总跑偏，希望有人接住选题、素材、排版这些环节。
- **知乎博主**：写长答案最怕事实硬伤和 AI 味，希望有人帮查引用、挑毛病、按你的风格写。
- **小红书博主**：要做封面、做图卡、精简文案，又要适配平台调性，希望团队一条龙出可发版本。
- **内容运营**：手上多篇内容要并行，每篇都要把关，希望有一套不会跳过关键步骤的流程化底座。
- **任何想把 AI 用成编辑部而不是单个写手的人**：你已经受够了「一个 AI 一口气写完五千字、方向跑偏只能全部推翻」。

## 作用是什么

这是一支专门帮你写新媒体文章的 AI 团队。你只需要跟秘书说一句话，比如「帮我写一篇关于 AI Agent 的公众号文章」，剩下的事团队会按流程一步步走——选题、查素材、搭骨架、落笔、审查、去 AI 味、排版分发，每个环节都有专门的角色负责。

它跟市面上的 AI 写作工具不一样的地方在于：AI 写作工具要么一口气写完五千字让你推翻重来，要么完全不动脑子让你写一句它改一句；而这支团队更像一家真有人分工的小编辑部——**8 个角色各管一摊，7 道关卡道道把关，每个关键节点都停下来等你拍板**。AI 不会一句话就开跑，也不会一口气写到黑。你只在关卡上做判断，环节内 AI 加速。

目标平台：微信公众号、知乎、小红书（可扩展至掘金、CSDN、Twitter）。

## 功能有哪些

你跟秘书说一句话，下面这些事情都能交给团队：

| 你的场景 | 团队怎么帮你 |
| --- | --- |
| 「最近不知道写啥」 | 选题官扫 40+ 平台热点，给你 3-5 张选题卡片，每张带 HKRT 质检评分（话题度/知识度/共鸣度/时效度） |
| 「有个话题但没素材」 | 研究员四维检索（个人笔记→站内→学术→全网），事实核查，重要事实至少 2 个独立信源 |
| 「素材有了但不知道怎么组织」 | 大纲师搭横纵骨架（深度模式专属），纵向时间脉络 + 横向竞品对比 + 未来三剧本 |
| 「写出来总是一股 AI 味」 | 三重保障去 AI 味：撰稿人按风格指纹写 + 审查员四层金字塔审查 + 风格官 9 维画像校准 |
| 「写完还要手动排版好几个平台」 | 排版师一键输出公众号/知乎/小红书三个版本，可直接复制粘贴发布 |
| 「AI 写的没有我的味道」 | 风格官分析你 3-5 篇代表作，提炼 9 维写作指纹注入撰稿人，越写越像你 |
| 「深度长文写不动」 | 深度模式（触发词「写深一点」），字数 5000-10000，横纵骨架加持 |

支持的 5 种文章类型，每种都有读者读完应能说的一句话：

| 类型 | 定位 | 读者看完应能说 |
| --- | --- | --- |
| case | 产品体验型 | 「原来是这样落地的」 |
| insight | 现象解读型 | 「本质原来是这样」 |
| lab | 调查实验型 | 「太有意思了，我也想试」 |
| review | 工具分享型 | 「我的场景应该用 X，因为…」 |
| tutorial | 方法论分享型 | 「照着走了一遍，真的做出来了」 |

8 个角色，每个只干一件事，不让一个人又写又查又排版：

| 编号 | 角色 | 干什么 | 通俗解释 |
| --- | --- | --- | --- |
| 00 | 秘书 / 调度官 | 唯一入口，接活、分派、盯 Gate、管预算 | 像「前台 + 项目经理」。你只跟它说话，它把活儿派给对的人 |
| 01 | 选题官 / 热点雷达 | 热点扫描、HKRT 质检、选题卡片 | 像「情报员」。每天扫 40+ 平台热榜，筛出值得写的 |
| 02 | 研究员 / 素材官 | 四维素材检索、事实核查、引用源台账 | 像「资料员」。每个事实至少 2 个独立信源交叉验证 |
| 03 | 大纲师 / 结构师 | 横纵骨架、SCQA/金字塔结构 | 像「建筑师」。深度模式才出场，搭好骨架再动笔 |
| 04 | 撰稿人 | 落正文，写一段确认一段，按风格路由 | 像「主笔」。不一口气代写，按你的风格指纹写 |
| 05 | 审查员 / 质检官 | 四层金字塔审查 + 全文去 AI 化 | 像「主编 + 校对」。只提问题不给答案，逼你自己想 |
| 06 | 风格官 / 风格指纹官 | 9 维写作指纹画像、画像注入、风格学习沉淀 | 像「你的数字替身」。你每次改稿它都学，越写越像你 |
| 07 | 排版师 / 分发官 | 多平台排版、可复制粘贴格式 | 像「美工 + 发行」。公众号/知乎/小红书三版本一键出 |

## 优势是什么

### 去 AI 味三重保障

读者和平台对 AI 味零容忍，团队从源头到终审贯穿三重保障：撰稿人动笔前先做四步定位（文体/读者/场景、作者站位、组织原则、首稿检查），按风格官注入的 9 维画像执行；审查员做四层金字塔审查（L1 硬规则扫描 → L2 风格一致性 → L3 内容质量深审 → L4 活人感终审），任一层不过即返工；风格官分析你 3-5 篇代表作，提炼 9 维写作指纹（开场句式、段落首句钩子率、类比库偏好、判断词、金句结构等），画像不是模糊评价而是机械可复现的量化指标。

### 七道 Gate 门控

每篇文章从选题到交付要过 7 道关卡：Gate 0 需求共识、Gate A1 素材确认、Gate A2 骨架拍板（深度模式）、Gate A 初稿确认、Gate B 审查确认、Gate C 去 AI 味确认、Gate E 终稿确认。**每道关卡都停下来等你拍板**，你不点头 AI 绝不往下走。这叫「防跳协议」——AI 每步输出前必须声明当前在第几步、下一步第几步、等你确认，遇到 Gate 立即停。违反防跳协议等于任务失败，从当前步骤重来。

### 三层 QA 框架

市面上大部分 AI 写作工具只做第一层挑错，团队做了三层：第一层是「文章没有错」——事实准、引用全、无 AI 味、合规范；第二层是「文章值得读」——读者花 10 分钟能得到什么，开头有没有钩子、中段有没有信息密度低谷、结尾读者能记住什么；第三层是「越写越像你」——每篇交付后评估强化了你的哪个独特视角、沉淀了什么可复用资产，把你的独有判断显性化。前两层可被短期投入复刻，第三层只能靠时间沉淀。

### 风格学习机制

你每次修改终稿后回复「继续」，审查员会逐段对比 AI 原稿与你修改版，识别所有调整，提炼 P0（≥3 次可复用）/ P1（2 次观察）/ P2（单次仅本篇）规律，移交风格官评估。P0 经你确认后写入风格库，写入前快照备份可回滚。下次写作时撰稿人按更新后的画像执行。**你改得越多，AI 越懂你**——这是去 AI 味战略的长期保障，因为 AI 味的根源之一就是作者认知投降后 Agent 全权代理。

## 特点是什么

- **有分工**：8 个角色各管一摊，秘书统一调度，每个角色只干一件事。
- **有 Gate**：7 道关卡道道把关，每个关键节点停下来等你拍板，防跳协议是最高优先级。
- **有 QA**：三层质量保障，从挑错到读者价值到长期竞争力层层递进，13 个方法论吸收覆盖三层 QA 执行准则。
- **有去 AI 味**：撰稿人按风格路由写 + 审查员四层金字塔审查 + 风格官 9 维指纹画像，三重保障从源头到终审。
- **有风格学习**：你每次改稿都被记录分析，P0/P1/P2 分级沉淀进风格库，越写越像你。
- **有预算护栏**：循环次数、时长、工具调用、花费都设上限，烧不动你的钱空转。

## 怎么用

**第一步：克隆仓库切到 writer-team 分支**

```bash
git clone -b writer-team <仓库地址>
cd digital-office-agent-system
```

**第二步：阅读 TEAM.md 了解团队配置**

`TEAM.md` 是团队总览文档，包含 8 角色分工、编排流程、Gate 体系、工作包规范、去 AI 味策略、QA 体系、技能索引、平台适配、文章类型、深度模式等完整信息。

图形界面可以用 `web-config` 配置端口、访问令牌、模型连接和内网访问策略，再用 `web-serve` 启动 Web UI 与 PWA。普通入口是 `/`，管理后台入口是 `/admin`；PWA 会缓存离线壳，适合内容团队在单机、内网服务器或中小企业轻量部署场景使用。

**第三步：告诉秘书你要写什么**

对秘书说一件事即可，秘书会先复述意图、追问 3 层（目标读者、价值承诺、不能妥协的边界），确认后才正式启动。比如「帮我写一篇关于 AI Agent 工作流的 insight，Kenny 风格，公众号 + 知乎」，秘书不会一句话就开跑。

**深度模式（可选）**：触发词「写深一点」「横纵分析」「深度文」「5000-10000 字」。开启后字数 3500-6000 → 5000-10000，素材升级四维检索，新增横纵骨架（纵向时间脉络 + 横向竞品对比 + 横纵交汇 + 未来三剧本），风格保持人味。

## 三个分支的关系

这个仓库有三个并列分支，各自服务不同场景，共享同一套工程底座：

| 分支 | 定位 | 适合谁 |
| --- | --- | --- |
| **main** | 数字办公系统（通用底座） | 通用企业办公，七角色齐全，覆盖产品/设计/研发/法务/写作全链路 |
| **research-team** | 科研团队特化 | 做科研的人，强调研究完整性门控、评分轨迹追踪、状态可观测 |
| **writer-team**（你现在看的这个） | 写作团队特化 | 做内容的人，8 角色写作流水线 + 去 AI 味 + 读者价值验证 |
