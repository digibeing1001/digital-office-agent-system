# Digital Office · 写作团队版

> 一支「有人负责、交代清楚、不瞎循环、关键步骤有人把关」的新媒体写作团队。
> 你拿小锤指方向，AI 拿大锤出力——你始终是那个师傅。

---

## 这是什么

一句话定位：**一支专门帮你写新媒体文章的 AI 团队**。

打个比方。你平时找人代写一篇文章，会遇到两种情况：要么对方一口气写完五千字，方向跑偏了你只能全部推翻重来；要么对方完全不动脑子，你写一句他改一句，最后还是你在写。

这支团队不是这样。它更像一家真有人分工的小编辑部——**8 个角色各管一摊**：有人专门盯热点选题，有人专门查素材，有人专门搭结构，有人专门落笔写，有人专门挑毛病，有人专门去 AI 味，有人专门维护你的写作风格，还有人专门排版分发。**你只需要跟秘书说一句话**，比如「帮我写一篇关于 AI Agent 的公众号文章」，剩下的事团队会按流程一步步走。

最关键的是：**每个关键节点都停下来等你拍板**。选题定了让你看一眼，素材齐了让你确认，初稿写完让你改，审查报告给你看，去完 AI 味再让你过目，终稿确认后才交付。**AI 不会一句话就开跑，也不会一口气写到黑**。

目标平台：微信公众号、知乎、小红书（可扩展至掘金、CSDN、Twitter）。

只要你写过公众号、知乎或小红书，下面这些场景你应该都遇到过：

| 「想写但没时间一篇篇盯」 | 你只在 7 个 Gate 拍板，环节内 AI 加速；预算护栏防止烧钱空转 |

## 八个写作角色

| 编号 | 角色 | 干什么 | 通俗解释 |
|---|---|---|---|
| 00 | 秘书 / 调度官 | 唯一入口，接活、分派、盯 Gate、管预算 | 像「前台 + 项目经理」。你只跟它说话，它把活儿派给对的人，盯着每道关卡别跳过 |
| 01 | 选题官 / 热点雷达 | 热点扫描、HKRT 质检、选题卡片 | 像「情报员」。每天扫微博/知乎/抖音/B站/36氪等 40+ 平台热榜，给你筛出值得写的 |
| 02 | 研究员 / 素材官 | 四维素材检索、事实核查、引用源台账 | 像「资料员」。个人笔记→站内→学术→全网层层搜，每个事实至少找 2 个独立信源交叉验证 |
| 03 | 大纲师 / 结构师 | 横纵骨架、SCQA/金字塔结构 | 像「建筑师」。深度模式才出场，给你搭好骨架再动笔，避免写到一半发现结构散了 |
| 04 | 撰稿人 | 落正文，写一段确认一段，按风格路由 | 像「主笔」。不一口气代写，写一段让你看一段，按你的风格指纹写，不是通用 AI 味 |
| 05 | 审查员 / 质检官 | 四层金字塔审查 + 全文去 AI 化 | 像「主编 + 校对」。只提问题不给答案（逼你自己想），然后逐段扫 AI 味打分 |
| 06 | 风格官 / 风格指纹官 | 9 维写作指纹画像、画像注入、风格学习沉淀 | 像「你的数字替身」。分析你的代表作提炼指纹，你每次改稿它都学，越写越像你 |
| 07 | 排版师 / 分发官 | 多平台排版、可复制粘贴格式 | 像「美工 + 发行」。公众号/知乎/小红书三个版本一键生成，直接复制粘贴就能发 |

> 单角色单标准：每个角色只干一件事，不让一个人又写又查又排版——那样谁都没法专注。

## 七道 Gate 门控

Gate 是「关卡」的意思。每篇文章从选题到交付，要过 7 道关卡，**每道关卡都停下来等你确认**，你不点头 AI 绝不往下走。这叫「防跳协议」——防止 AI 跳过关键步骤自己往下跑。

| Gate | 关卡名 | 防什么 | 你要做什么 |
|---|---|---|---|
| Gate 0 | 需求共识 | 防止 AI 理解错你的意图就开跑 | 看秘书复述的需求摘要 + HKRT 评级，确认对路 |
| Gate A1 | 素材确认 | 防止动笔前素材没找齐 | 看素材清单，确认够用再动笔 |
| Gate A2 | 骨架拍板 | 防止深度长文结构散了（仅深度模式） | 看横纵骨架，拍板结构再写 |
| Gate A | 初稿确认 | 防止 AI 一口气写完才发现方向跑偏 | 看初稿，改完回「继续」才进审查 |
| Gate B | 审查确认 | 防止审查报告敷衍了事 | 看完整审查报告 L1-L4，确认修改意见 |
| Gate C | 去 AI 味确认 | 防止「处理完成」一句话带过 | 看处理摘要（高频/中频/低频各几处），确认 |
| Gate E | 终稿确认 | 防止没确认就交付 | 看终稿 + 引用台账，回「继续」才触发风格学习 |

> 防跳协议是最高优先级。AI 每步输出前必须声明「当前在第 X 步，下一步第 Y 步，等你确认」。遇到 Gate 立即停，你没回话之前禁止输出下一步任何内容。违反 = 任务失败，从当前步骤重来。

## 三层 QA 框架

QA 是「质量保障」的意思。市面上大部分 AI 写作工具只做第一层（挑错），我们做了三层。这是这支团队区别于其他 AI 写作工具的核心能力。

> 灵感来自吴恩达的「三层 Loop」理论：第一层是分钟级的自动迭代（基础但不是核心竞争力），第二层是小时级的人介入判断（核心竞争力），第三层是天/周级的市场反馈（决定生死）。

### 第一层 · 执行层：文章「没有错」

**目标**：事实准、引用全、无 AI 味、合规范。

**通俗解释**：这是「挑错型」QA。审查员的四层金字塔审查 + 去 AI 味三重保障都在这一层。文章不能有事实硬伤、不能漏引用、不能一股 AI 味。

**触发节点**：Gate A1（素材确认）/ Gate B（审查确认）/ Gate C（去 AI 味确认）

### 第二层 · 读者价值层：文章「值得读」

**目标**：读者花 10 分钟读完，能得到什么？这个回报对得起他的时间吗？

**通俗解释**：第一层问「这篇文章有没有错」，第二层问「读者为什么要读完」。审查员不只挑错，还要做「读者代言人」——开头 100 字有没有钩子？中段有没有连续 800 字信息密度低谷？结尾读者合上文章能记住什么？

**触发节点**：Gate 0（需求共识，秘书定义读者画像 + 价值承诺）/ Gate A（初稿确认，审查员验证价值兑现）/ Gate E（终稿确认，平台适配验证）

### 第三层 · 长期竞争力层：「强化作者辨识度 + 积累复利资产」

**目标**：构建别人无法短期复刻的上下文优势。

**通俗解释**：前两层可以被短期投入复刻，第三层只能靠时间沉淀。每篇文章交付后，风格官评估「这篇文章强化了你的哪个独特视角」「沉淀了什么可复用的认知/方法/案例」。你的每一次 Gate 拍板决策、每一次终稿修改理由，都作为「上下文资产」显式归档，让 AI 越来越懂你。

**触发节点**：Gate E 后（终稿交付后）+ 季度复盘

> 「上下文优势」替换「品味」概念（来自吴恩达原文）：品味暗示不可传授，上下文优势指向可操作路径——把你的独有判断显性化，AI 就能变好。

## 去 AI 味三重保障

这是这支团队的核心卖点。读者和平台对 AI 味零容忍，我们从源头到终审贯穿三重保障。

### 保障 1：撰稿人按风格路由（不是通用 AI 写作）

撰稿人动笔前必须先做四步（来自 humanize-chinese-writing 方法论）：
1. 确定文本任务（文体/读者/场景）
2. 建立作者站位（正文脱离聊天上下文独立成立，不保留「好的我来帮你写」之类的回答者惯性）
3. 选择组织原则（按文体选主线）
4. 写出并检查首稿

并且按风格官注入的 9 维画像执行，不是通用 AI 凭感觉写。

### 保障 2：审查员四层金字塔审查 + 33 模式 AI 味检测

四层金字塔审查（逐层推进，任一层不过即返工）：

| 层级 | 检查什么 | 通过标准 |
|---|---|---|
| L1 硬规则扫描 | 禁用词、AI 伪口语、禁用标点、结构套话、空泛工具名 | 五项扫描零命中 |
| L2 风格一致性 | 开头是否场景切入、长短句交替、回环呼应、知识输出方式 | 开头必须通过，其余至少 3/4 通过 |
| L3 内容质量深审 | 观点支撑、对立面与同理心、文化升维、非技术可读性、读者锚点、事实性 | 观点支撑和事实性必须全过 |
| L4 活人感终审 | 一个判定问题代替一堆规则 | 任何一段 AI 味重即返工 |

L4 活人感终审的核心判定问题：

> 读完这篇文章，是一个有见识的朋友在跟我聊一件打动他的事，还是 AI 在给我输出信息？

### 保障 3：风格官 9 维指纹画像 + Voice Calibration 机械扫描

风格官分析你 3-5 篇过往代表作，提炼 9 维写作指纹：

| 维度 | 说明 | 示例 |
|---|---|---|
| 开场句式偏好 | 习惯怎么开头 | 「故事是这样的」+ 真实场景 |
| 段落首句钩子率 | 段首是否能钩住读者 | 量化百分比 |
| 类比库偏好 | 偏好哪类跨领域类比 | 生活化类比解释技术概念 |
| 场景案例密度 | 每 300 字必有一个生活场景案例 | 量化 |
| 口头禅/判断词 | 高频使用的判断词 | 「我自己私心觉得」「我非常不推荐」 |
| 金句结构 | 习惯的金句样式 | 短句 + 对比结构 + 动作动词（≤25 字） |
| 思考路径 | 从什么视角切入 | 产品经理视角 / 实践者视角 |
| 价值判断偏好 | 习惯做什么判断 | 反共识 / 谨慎保守 / 案例先行 |
| 节奏偏好 | 句长变化模式 | 长短句交替 + 一句话独立成段 |

> 画像不是「文风活泼」这种模糊评价，而是机械可复现的量化指标（句长分布、虚词频率、段首钩子率等）。画像注入撰稿人后，AI 按你的指纹写，不是通用 AI 味。

### 风格学习机制（越写越像你）

你每次修改终稿后回复「继续」，审查员会：
1. 逐段对比 AI 原稿与你修改版，识别所有调整
2. 提炼 P0（≥3 次可复用）/ P1（2 次观察）/ P2（单次仅本篇）规律
3. 移交风格官评估，P0 经你确认后写入风格库（写入前快照备份可回滚）

下次写作时，撰稿人按更新后的画像执行。**你改得越多，AI 越懂你**。

## 十三个方法论

这 13 个方法论来自 arxiv 论文 + GitHub 工具调研 + 业界实践，全部以「方法论参考」形式集成（不直接下载代码，因为团队是 Markdown-only 架构）。每个方法论解决一个具体的写作问题。

### 第一层 QA（执行层）吸收的 4 个

| # | 方法论 | 解决什么写作问题 |
|---|---|---|
| 1 | LLM-as-judge 三类偏见消除 | AI 自己评自己会偏心——偏好特定位置、偏好更长文本、偏好同源模型输出。强制要求写作和审查的 AI 来自不同模型家族，盲审、多样本投票 |
| 2 | FActScore 原子化事实核查 | 审查员不能笼统说「这篇文章有问题」，要把文章拆成一条条原子事实，每条标注：已验证（≥2 信源）/ 单一信源 / 错误存疑 |
| 3 | CRITIC 工具增强审查 | 事实类声明用搜索工具查证（不靠模型脑补），数据类声明用计算工具验证，逻辑类推理用代码验证 |
| 4 | Evals 可演进 | 检查清单不是固定不变的——踩过的坑固化成新检查项，定期清理失效的规则。改 prompt 时跑评估矩阵对比前后，防止「修了 AI 味、坏了事实性」 |

### 第二层 QA（读者价值层）吸收的 4 个

| # | 方法论 | 解决什么写作问题 |
|---|---|---|
| 5 | MT-Bench 多轮体验衰减 | 读者价值不能只看开头钩子。做三段式体验扫描：开头钩子是否有效、中段信息密度是否持续、结尾读者能记住什么 |
| 6 | HHH 三维价值（Helpful/Honest/Harmless） | 读者价值 = 有用 × 诚实 × 无害，任一为 0 总分归零。有用 = 有信息增量；诚实 = 不夸大不误导；无害 = 不焦虑诱导、不制造对立 |
| 7 | 多 Agent 辩论收敛 | 读者价值不是单人判定。审查员（质检视角）+ 秘书（PM 视角）+ 作者（品味视角）+ 模拟读者（读者视角）四视角看同一篇，反对意见必须显式记录 |
| 8 | 个性化评判标准 | 给专业读者和给大众读者的评判标准不同。Gate 0 声明目标读者画像后，动态调整评判标准——专业读者信息密度阈值上调，大众读者开头钩子权重上调 |

### 第三层 QA（长期竞争力层）吸收的 5 个

| # | 方法论 | 解决什么写作问题 |
|---|---|---|
| 9 | Reflexion 情景记忆 | 每篇文章交付后生成「踩坑反思」——哪些事实错误是审查员漏检的？哪些 AI 味模式是新发现的？下一篇启动时审查员先读取最近反思 |
| 10 | Self-Rewarding 自我进化 | 撰稿人/审查员/风格官各自做自评——这篇初稿哪里自己不满意？四层审查哪些判断事后看是错的？画像注入对成文的贡献度几何？ |
| 11 | 风格指纹量化（StyleLLM） | 风格不能靠感觉。把风格量化为可计算的指纹（句长分布/虚词频率/标点偏好/语义模式），每篇交付后计算与画像基线的偏离度——是你在迭代风格，还是 AI 偏离了画像？ |
| 12 | ExpertPrompting 身份条件 | 审查员不应以「通用质检员」身份审查，应以「目标读者画像」身份审查。审查一篇给产品经理看的文章时，审查员代入产品经理视角 |
| 13 | 认知投降检测 | 监测作者是否陷入「认知投降」——越来越快点通过 Gate、越来越少提修改意见、越来越多「你看着办」。一旦触发，强制作者深度介入下 2-3 篇做判断校准 |

> 第 13 条是去 AI 味战略的长期保障。AI 味的根源之一就是作者认知投降后 Agent 全权代理——你以为省心了，其实把判断权让渡给了 AI，文章就慢慢变成 AI 味了。

## 风格库

### kenny-writer.md（卡兹克风格指纹）

核心资产。来源：卡兹克开源 Skill（GitHub 7.7k Star）+ 三篇代表作逐段分析。包含：

- 9 维风格指纹画像（开场句式、段落首句钩子率、类比库偏好、案例密度、判断词、人称、金句结构、收尾、古诗词收尾）
- 开头技法（场景切入型 / 颠覆结论型 / 反常事实型 / 痛点共鸣型）
- 节奏技法（对话体叙事 + 一句话独立成段 + 长短句交替）
- 降维技法（场景 → 跨领域类比 → 抽象框架，三步走）
- 金句设计原则（≤25 字 + 对比结构 + 动作动词 + 口语化 + 反共识 + 可独立转发）
- 收尾技法（下一步动作 + 软引流 + 情感落点，禁止「欢迎点赞转发」）
- 叠甲技法（强观点前后自我消解 + 主动暴露失败，消除「专家训话感」）
- AI 占比边界（教程 0% / 论文解读 15-20% / 观点+案例 30-40%）
- 禁用清单（禁用词、AI 伪口语、禁用标点、禁用结构、空泛工具名）
- 四层自检体系（L1 硬规则 / L2 风格一致性 / L3 内容质量 / L4 活人感终审）
- 风格学习机制（P0/P1/P2 分级 + 写入前快照可回滚）

## 与其他分支的关系

`digital-office-agent-system` 仓库下有三个并列分支，互不替代、互不背离：

| 分支 | 定位 | 核心能力 |
|---|---|---|
| **main** | 数字办公系统（通用底座） | 三层员工模型：秘书控制面 → 数字员工 → 技能 staff lanes。通用数字办公底座，可承载各类 Agent Team |
| **research-team** | 科研团队特化 | 科研秘书 + PI + 工程师的科研协作系统。解耦 judge 防自评偏差、scoring_trajectory 趋势跟踪 |
| **writer-team** | 写作团队特化（本分支） | 8 角色 + 7 Gate + 三层 QA + 去 AI 味三重保障。专门写新媒体文章 |

三方在不背离各自设计目标的前提下相互学习：

- main → writer-team：提供 SOUL 模板（身份/思维方式/禁区结构）+ taste-skill（反 AI Slop，14 子技能，视觉层去 AI 味）
- research-team → writer-team：提供解耦 judge（模型家族隔离 + 盲审 + 多样本投票）+ scoring_trajectory（趋势/波动度/停滞/回退四指标）+ SOUL 模式
- writer-team → main / research-team：反向输出三层 QA 框架 + Gate 防跳协议

## 怎么用

### 三步上手

**第一步：克隆仓库切到 writer-team 分支**

```bash
git clone -b writer-team <仓库地址>
cd digital-office-agent-system
```

**第二步：阅读 TEAM.md 了解团队配置**

TEAM.md 是团队总览文档（307 行），包含 8 角色分工、编排流程、Gate 体系、工作包规范、去 AI 味策略、QA 体系、技能索引、平台适配、文章类型、深度模式等完整信息。

**第三步：告诉秘书你要写什么**

对秘书说一件事即可，秘书会先复述意图、追问 3 层，确认后才正式启动。不会一句话就开跑。

### 示例对话

**场景一：带选题直进**

```
你：帮我写一篇关于 AI Agent 工作流的 insight，Kenny 风格，公众号 + 知乎。

秘书：[复述意图] 你想写一篇 insight 型文章，主题是 AI Agent 工作流，
      用卡兹克风格，发公众号和知乎两个平台。对吗？
      我追问三层：
      1. 目标读者是谁？（产品经理？开发者？路人读者？）
      2. 读者花 10 分钟读完能得到什么？（一句话价值承诺）
      3. 有没有不能妥协的边界？（字数上限？必须提到的案例？）

你：[回答追问]

秘书：[需求共识摘要 + HKRT 评级] —— Gate 0，等你确认。
```

**场景二：没选题，让团队帮你找**

```
你：最近 AI 圈有啥可写的？

秘书：[路由判断] 你没带选题，我派选题官扫热点雷达。
      [3-5 张热点卡片，每张带 HKRT 评分] —— 你选一张，我们继续。
```

**场景三：有素材，让团队帮你写成文章**

```
你：把这份素材写成 case。

秘书：[素材契约激活] 我派研究员确认素材契约（主旨锁、保留率、AI 占比上限），
      确认后进入 Step 3 四维素材检索补充。
```

### 深度模式（可选）

触发词：「写深一点」「横纵分析」「深度文」「5000-10000 字」。

开启后：字数 3500-6000 → 5000-10000；素材升级四维；新增 Step 3.5 横纵骨架（纵向时间脉络 + 横向竞品对比 + 横纵交汇 + 未来三剧本）；风格保持人味。
