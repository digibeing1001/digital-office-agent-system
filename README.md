# Digital Office 数字办公室

> 你只管把事情说清楚，秘书负责找人、盯进度、管审批；研究、产品、设计、开发、写作、规划、法务，各有一位对结果负责的数字员工。

## 给谁用

只要你想让一支 AI 团队帮你干活，这套系统就适合你。下面这几类人尤其合适：

- **创业者**：一个人扛一摊事，缺人手又不想再雇一个团队，希望 AI 把杂事接过去。
- **产品经理**：手上需求多、版本乱，需要有人帮忙从一句话想法拉到能交付的 PRD，再到上线检查。
- **独立开发者**：能写代码但不想一个人卡在调研、设计、文档这些环节上，希望有人分工。
- **小团队负责人**：团队几个人，但流程乱、上下文丢、关键动作没人把关，希望有个能管住事的底座。
- **任何想把 AI 用成团队而不是单兵的人**：你已经受够了「一个 AI 什么都干、什么都干一半」的状态。

## 作用是什么

Digital Office 把一家有分工的数字公司装进你电脑里。你不用记每个数字员工叫什么、擅长什么——你只跟秘书说一句话，秘书会先确认你到底要什么，再把活儿分给最合适的那位数字员工，做完之后把结果连同过程记录一起交回给你。

它跟市面上的 AI 助手不一样的地方在于：市面上的 AI 像一个什么都能干一点的临时工，你交代一件它做一件，下一件就忘了上一件；而 Digital Office 更像一家真实的小公司——**每件事只有一个负责人、每一步都有记录可回看、关键动作有人把关、任务不会自己无限循环烧你的钱**。它不追求「比别人更聪明」，而是让你随时知道事情现在在哪里。

## 功能有哪些

你跟秘书说一句话，下面这些事情都能交给数字员工：

| 功能 | 数字员工会做什么 |
| --- | --- |
| 做调研 | 查同类产品、找市场数据、整理带出处的证据，给一份能直接拿去开会的研究报告 |
| 写需求文档 | 把「我想做一个 XX」这种模糊想法变成清晰 PRD，列好优先级和验收标准 |
| 排项目计划 | 把大目标拆成几步、定里程碑、标依赖和风险，让你知道先做什么后做什么 |
| 设计界面 | 从用户流程到视觉稿画好原型，做无障碍检查，给程序员一份能照着实现的稿子 |
| 写代码 | 实现新功能、修 Bug、跑测试和构建验证，交付前过质检门禁，不是「能跑就行」 |
| 写文章文案 | 把零散材料整理成文章、产品文案、发布说明、用户文档，做事实核对和润色 |
| 审合同查合规 | 审供应商合同、隐私条款、产品合规风险，给企业内部审查草稿（外发仍需人类律师把关） |
| 串联多步 | 一句话串起调研→需求→设计→开发这条流水线，秘书自动安排接力 |

这家公司一共有七位数字员工，你不用记名字，秘书会自动判断该找谁：

| 数字员工 | 负责什么 | 主要本事 |
| --- | --- | --- |
| 秘书 | 接活、确认意图、分派、盯进度、管审批 | 团队排班、找技能、资料运营 |
| 研究员 | 查资料、找证据、写引用、做调研 | 迭代检索、深度研究、市场调研 |
| 产品经理 | 把模糊需求变清晰任务、排优先级、定验收 | 需求澄清、PRD 起草 |
| 规划师 | 拆项目、排时间线、标依赖和风险 | 头脑风暴、写计划 |
| 设计师 | 界面流程、视觉、原型、无障碍 | 设计、设计质检 |
| 程序员 | 写代码、调试、测试、部署准备 | 编码质检、测试驱动、编码规范 |
| 写作员 | 提纲、起草、编辑、终稿 | 写作、文案、内容策略 |
| 数字律师 | 合同、隐私、产品合规、用工与知识产权、争议分流 | 法务工作流（按通道干活） |

每位数字员工对一个方向负责。**不会出现三四个 AI 抢同一件事**，也不会层层上报走流程。

## 优势是什么

### 一个秘书入口，七位数字员工

你只需要对秘书说一件事，秘书判断该找谁、要不要先问清楚、哪些步骤要审批，然后把活儿交给一位明确的数字员工。**每件事只有一个负责人**，不会出现「三四个 AI 抢同一件事」的混乱。

### 四步干活法，不会无限循环

每位数字员工干活都按「搞清楚 → 定路线 → 去执行 → 看结果」四步走，每一步都记下来可以回看。关键是**后台控制系统说了算，AI 只能建议**——循环次数、重试、时长、工具调用、花费都有上限，进度停滞会被发现，任务不会无限自我反思烧你的钱。

### 任务交接靠带版本的「工作包」

任务从一个数字员工转给下一个时，不会把聊天记录一股脑塞过去，而是生成一个紧凑、带版本、可校验的工作包，写清楚目标、限制、验收标准、事实与假设、证据位置、决定、交接印记。接收方必须明确点「收到」，发送方才算干完。**不靠聊天记忆，不丢上下文**。

### 关键动作有人把关

不是什么动作都能让 AI 自己决定。系统列出 **11 类硬管控动作**——对外发送、公开发布、上线部署、删除、覆盖、付款、数据导出、记入长期记忆、晋升为公司知识、给出受监管的专业建议、最终交付——超过风险门槛就暂停请人确认。**AI 可以请求暂停，但只有后台控制系统能冻结和恢复**。

## 特点是什么

- **有分工**：七位数字员工各管一摊，秘书统一调度，每件事只有一个负责人。
- **有流程**：四步干活法贯穿每位员工，从搞清楚到看结果，每步可回看。
- **有交接**：任务交接靠带版本的工作包，不靠聊天记忆，接收方必须明确收到。
- **有门控**：11 类关键动作 AI 不能自己拍板，超阈值自动暂停请人确认。
- **有预算护栏**：循环次数、时长、工具调用、模型调用、花费都设上限，烧不动你的钱。
- **有审批**：风险动作走人工审批，认知投降防御要求审批人写不少于 30 字理解总结才能放行。
- **可回看**：每次选了哪条路、调用了多少、产出了什么、有没有失败，全部写进记录。
- **不瞎循环**：AI 可以提下一步，但只有后台控制系统能在状态间切换，进度停滞会被发现。

## 怎么用

**第一步：一条命令装好**（在 WSL 或 Linux 终端运行）

```bash
curl -fsSL https://raw.githubusercontent.com/digibeing1001/digital-office-agent-system/main/install.sh | bash
```

装到 `~/.hermes`，会自动跑健康检查和发布质检。以后升级用 `~/.hermes/update`，只更新软件和内置技能，不动你的项目、资料库、任务、审批和个人偏好。

**第二步：启动界面**

```bash
digital-office-gui        # 普通用户界面
digital-office-gui --admin  # 管理后台
```

系统优先用 `http://127.0.0.1:8787/`，端口被占就自动找下一个可用的。要让别的电脑访问必须配访问令牌，不会裸露在局域网上。

图形界面也可以直接用命令启动：`web-config` 用来配置端口、访问令牌和模型连接，`web-serve` 用来启动 Web UI 和 PWA；普通入口是 `/`，管理后台入口是 `/admin`。PWA 会缓存离线壳，适合中小企业在内网、单机或轻量服务器上部署。

**第三步：跟秘书说话**

直接对秘书说一件事，比如「先调研同类产品，再整理需求，设计界面，最后实现一个能跑的前端版本」，秘书会先复述对你意图的理解、问你至少三个层层深入的问题，等你确认底稿之后正式工作才会启动。

## 三个分支的关系

这个仓库有三个并列分支，各自服务不同场景，共享同一套工程底座：

| 分支 | 定位 | 适合谁 |
| --- | --- | --- |
| **main**（你现在看的这个） | 数字办公系统 | 通用企业办公，七角色齐全，覆盖产品/设计/研发/法务/写作全链路 |
| **research-team** | 科研团队特化 | 做科研的人，强调研究完整性门控、评分轨迹追踪、状态可观测 |
| **writer-team** | 写作团队特化 | 做内容的人，8 角色写作流水线 + 去 AI 味、读者价值验证 |
