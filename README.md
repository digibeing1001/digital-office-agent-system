# Digital Office · 科研团队版

> 一支能查文献、定方向、做实验、写论文、审稿的 AI 科研队伍，从立项到交付全程陪你跑。

---

## 给谁用

- **研究生 / 博士**：一个人扛一个方向，文献看不过来、实验跑不出来、论文心里没底
- **青年学者 / 博士后**：没有大团队支撑，但科研全流程都得自己跑
- **科研助理**：帮导师跟进多个课题，需要工具帮你追踪进度、沉淀经验
- **跨学科研究者**：新进入一个领域，需要快速摸清前沿、找到研究空白

---

## 作用是什么

把一个完整的科研团队装进你的电脑——有人查文献、有人定方向、有人设计实验、有人跑代码、有人写论文、有人像审稿人一样挑毛病。你只跟科研秘书说一句话，剩下的事团队按流程接力干完。

跟普通 AI 工具最大的不同：**普通 AI 答完就散了；这套系统会记住踩过的坑、你纠正过的错误，下一个项目带着经验继续干，越用越顺手。**

## 功能有哪些

### 全流程覆盖

| 你遇到的问题 | 团队怎么帮你 |
|---|---|
| 「想研究 XXX，不知道从哪下手」 | 课题规划师把模糊想法拆成具体研究问题，定方向、找创新点、画路线图 |
| 「文献太多看不过来」 | 文献研究员从 arXiv / Google Scholar / Semantic Scholar 三源检索，PRISMA 流程筛选，每条结论追溯到原文 |
| 「实验怎么设计才站得住脚」 | 方法学专家选统计方法、算样本量、做因果推断，保证有对照、有显著性、能复现 |
| 「代码跑不出结果 / 不可复现」 | 实现工程师用 MLflow 记录每次实验，自动导出可复现包（代码+数据+配置+种子+环境） |
| 「数据不知道怎么画图」 | 数据分析师做统计检验、画出版级配图（Nature/Science/IEEE 风格）、核对每个数字 |
| 「论文写完怕被审稿人挑毛病」 | 质检员用顶会审稿标准模拟同行评审，把论文拆成一条条 claim 逐个验证 |
| 「怕引用了被撤稿的论文」 | 伦理员用三个源头交叉检查撤稿标记，用 JPlag 查代码查重，用 MinHash 查文本查重 |
| 「文献天天有新的，追不过来」 | 资料员每天自动抓新论文，去重入库打标签 |

### 十个科研角色

| 角色 | 干什么 | 什么时候登场 |
|---|---|---|
| 科研秘书 | 接活、分活、盯进度、评分验收 | 一直在，所有项目从这里开始 |
| 课题规划师（PI） | 定研究方向、找创新点、画路线图 | 立项阶段主力 |
| 文献研究员 | 查论文、写综述、找研究空白 | 文献阶段主力 |
| 学术写作员 | 写论文、改稿、排版 | 写作阶段主力 |
| 方法学专家 | 设计实验、选统计方法、算样本量 | 需要实验设计时 |
| 实现工程师 | 写代码、跑实验、封装可复现包 | 需要跑实验时 |
| 数据分析师 | 算数据、画图、核对数字 | 需要数据分析时 |
| 质检员 | 模拟同行评审、找方法学问题 | 论文初稿出来后 |
| 伦理员 | 查重、查合规、查造假、查署名 | 交付前最后审查 |
| 资料员 | 定时抓新论文、自动入库、建引文网络 | 后台默默跑 |

### 十个科研专用能力

| 能力 | 通俗解释 |
|---|---|
| 全链路可观测性 | 每次操作都有 trace 记录，随时能查「谁干的、调了什么、花了多少、有没有失败」 |
| 科学文献高精度问答 | 拿论文问问题，会检查有没有被撤稿、跟其他论文有没有矛盾 |
| 树状递归深度研究 | 把课题拆成一棵树，每个分支并行去查，最多搜 100 个来源 |
| 多样本投票验证 | 重要结论让 AI 独立采样 3 次，再找独立 judge 比对，多数一致才算数 |
| GOAP 目标规划 | 自动把大目标拆成小任务，走不通就重新规划路线 |
| 反思循环 | 失败了先反思「哪里不够、为什么、怎么改」，带着反思重写 |
| 综述生成 | 用多个视角互相提问发现研究角度，再写大纲和全文，每条引用都真的存在 |
| 撤稿专项检查 | 三个源头交叉检查引用的论文有没有被撤稿，核心论点依赖被撤稿论文直接打回 |
| 跨模型交叉验证 | 关键产出找 4 个不同模型家族分别评分，再找第三方盲审仲裁 |
| Socratic 引导立项 | 不立刻开干，而是从 5 个维度追问你的真实意图，问清楚了再动手 |

## 优势是什么

### 1. 反思进化闭环——越用越顺手

做得不好能返工，返工能反思，反思能沉淀，沉淀能复用。

- **评分不合格**：先生成反思（哪里不够、为什么、怎么改），带着反思重写，不是重新掷骰子
- **你纠正过**：判断是通用偏好还是一次性指正，通用偏好永久记住，一次性只存当前项目
- **项目跑完**：回顾全过程抽取经验，去重存入经验库，下个项目带着经验干活
- **实验失败**：即时归档失效条件和方法边界，不当成 anti-pattern 笼统处理

### 2. 七道科研完整性门控——AI 翻车的地方都设了硬关卡

| 门控 | 防什么 | 失败怎么办 |
|---|---|---|
| 幻觉引用阻断 | 编造不存在的文献 | 返工，最多 2 次 |
| 方法伪造阻断 | 方法步骤对不上代码 | 返工，最多 2 次 |
| 框架锁定阻断 | 跟已有工作撞车 | 返工，最多 2 次 |
| **数据伪造阻断** | 数字对不上原始数据 | **直接 fail，不可返工** |
| 过度泛化阻断 | claim 吹得太大 | 返工，最多 2 次 |
| 撤稿来源阻断 | 依赖被撤稿的论文 | 返工，最多 2 次 |
| 可复现性缺口阻断 | 别人复现不出来 | 返工，最多 2 次 |

**数据伪造零容忍**，因为数据造假是科研伦理红线——一旦坐实，论文撤稿、声誉扫地。宁可误杀，不可放过。

### 3. 六维评分卡——不是自己说做完就算数

满分 100 分，合格 75 分。严谨性 / 新颖性 / 可行性三项任一低于 50% 直接打回（一票否决）。

### 4. 人类介入决策——灰色地带不替你拍板

6 个条件命中就暂停流水线请你裁决：灰色区间得分、低置信度、审稿人分歧、claim 边界、方向性决策、重复返工预警。决策包给齐争议点、两方观点、可选方案和后果，**帮你梳理清楚再请你拍板，不是把难题原样抛给你**。

## 特点是什么

- **有分工**：10 个角色各管一摊，不是一个人又查文献又写论文又审稿
- **有流程**：从立项到交付按固定链路接力，每步有交接契约，不瞎跑
- **有门控**：7 道完整性关卡，数据伪造零容忍，其余最多返工 2 次
- **有记忆**：三层记忆（单会话反思 / 跨会话经验 / 失败模式库），越用越懂你
- **有评分**：六维评分卡，三项一票否决，不是自己说做完就算数
- **有人把关**：灰色地带主动请你裁决，不耗到返工上限才上报
- **可复现**：每次实验自动导出可复现包，别人照着能跑出一样的数字

## 怎么用

### 第一步：装工具

```bash
# 装 Digital Office 主体（切到 research-team 分支）
curl -fsSL https://raw.githubusercontent.com/digibeing1001/digital-office-agent-system/main/update | bash

# 装科研专用 Python 库
pip install arxiv scholarly semanticscholar pdfplumber \
            dowhy econml pyDOE2 statsmodels \
            mlflow optuna hydra-core stable-baselines3 \
            SciencePlots altair plotly seaborn \
            crossrefapi Pylatex latexify bibtexparser \
            factscore \
            feedparser paddleocr habanero datasketch \
            pybibliometrics knowledge-storm
```

GROBID（PDF 全文解析）和 JPlag（代码查重）需单独部署，看各自 GitHub 仓库说明。

### 第二步：跟科研秘书说一句话

```text
帮我查一下 LLM Agent 在科研领域的最新进展，出一篇综述。
```

```text
我想研究 AI 记忆框架，帮我从立项到实验设计走一遍。
```

```text
我有一份论文初稿，帮我模拟同行评审找问题。
```

秘书不会立刻动手，会先用 Socratic 引导式追问把意图问清楚：要写综述、提新方法、还是复现？给谁看？验收标准？有没有硬限制？答完就分给对应角色开干。
