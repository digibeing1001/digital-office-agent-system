# 数字生命 1001 · 新媒体写作团队

> 一支「有人负责、交代清楚、不瞎循环、关键步骤有人把关」的新媒体写作团队。
> 目标平台：微信公众号、知乎、小红书。可扩展至掘金/CSDN/Twitter。
> 产出标准：既吸引读者、获得流量，又体现专业性；终稿可直接复制粘贴发布，无 AI 味。

---

## 一、团队定位

把 `digital-life-1001` 写作 Skill 体系，落成一支可调度、可回看、可迭代的多 Agent 团队。每篇文章从「选题 → 素材 → 初稿 → 审查 → 去 AI 味 → 终稿 → 多平台排版」全流程，**人工在每个 Gate 把关，AI 在环节内加速**。

**核心目标**：
- 写出读者愿意读完、愿意转发的文章（流量）
- 体现专业深度与独家视角（专业性）
- 全文无 AI 味，像真人作者出品（人味，绝对要求）
- 终稿按平台排版好，可直接复制粘贴发布（可交付）

## 四、编排流程与 Gate 体系

```
你
 └─ 秘书（接活 + 确认意图）
     │
     ├─ [带选题] ────────────────────────┐
     │                                    │
     ├─ [未带选题] → 选题官(Step 0.5 热点雷达) ─┘
     │
     ▼
  Step 1 需求收集（秘书主持，7项必填）── HKRT 质检 ── Gate 0（需求共识确认）
     │
     ▼
  Step 2 加载子技能（类型 case/insight/lab/review/tutorial）
     │  └─ 风格官·建立/更新画像，注入撰稿人（无独立 Gate，被调用即执行）
     │
     ▼
  Step 3 研究员·四维素材检索 ── 素材确认 ── Gate A1（素材确认）
     │
     ├─ [深度模式] → Step 3.5 大纲师·横纵骨架 ── Gate A2（骨架拍板）
     │
     ▼
  Step 4 撰稿人·生成初稿（按风格库 + 风格官注入的画像）── Step 5 保存初稿 ── Gate A（初稿确认）
     │
     ▼
  Step 6 审查员·四层金字塔审查 ── Gate B（审查确认，不通过打回撰稿人）
     │
     ▼
  Step 7 审查员·全文去 AI 化（同一角色第二阶段） ── Gate C（去 AI 味确认）
     │
     ▼
  Step 8 保存终稿 + 引用台账 ── Gate E（终稿确认）
     │  └─ 审查员触发风格学习（提炼 P0/P1/P2 移交风格官）
     │     └─ 风格官·评估写入风格库（无独立 Gate，经用户确认后写入）
     │
     ▼
  Step 10 排版师·多平台排版 ── Step 11 输出链接 ── Step 12 发布准备
     │
     ▼
  ✅ 交付：初稿链接 + 终稿链接 + 各平台版本（可直接复制粘贴）
```

**6 个 Gate（人工把关，防跳协议）**：
- **Gate 0**：需求共识摘要（HKRT 通过后）
- **Gate A1**：素材清单确认（动笔前）
- **Gate A2**：横纵骨架拍板（仅深度模式）
- **Gate A**：初稿确认 + 字数
- **Gate B**：审查报告（必须完整输出 L1-L4，禁止「审查通过」一句话带过）
- **Gate C**：去 AI 味处理摘要（高频/中频/低频各几处，禁止「处理完成」带过）
- **Gate E**：终稿底稿 + 引用台账

**风格官无独立 Gate**：被秘书在 Step 2 调用建立画像、在 Step 8 后接收审查员移交的风格学习成果评估写入。两处都需经用户确认但不独立卡 Gate。

**防跳协议（最高优先级）**：每步输出前必须声明 `[当前: Step X · 下一步: Step Y（等待用户确认）]`；遇 Gate 立即停止，用户未确认前禁止输出下一步任何内容；违反 = 任务失败，从当前步骤重来。

## 五、工作包交接规范（不靠聊天记忆）

任务从一个 Agent 转给下一个时，生成**带版本、可校验的工作包**：

```
📦 工作包 v{N} · {from} → {to}
─── 目标 ───
真正想做成什么 / 当前目标 / 限制 / 验收标准
─── 事实与证据 ───
确定的事实 / 假设 / 不确定或冲突（各标可信度）
证据和产物放在哪里（按引用取回，不重复传大文件）
─── 决定 ───
已做过哪些决定 / 哪些内容被省略及如何找回
─── 交接印记 ───
交给谁 / 对方是否核对 / 是否要求补充信息
─── 主旨锁 ───
「主旨锁: {一句话复述素材核心主旨与情绪基调}」（写作全程对齐）
─── 素材契约 ───
素材保留率 ≥ 70% / 替换需声明 / AI 占比上限
```

**硬约束**：AI 的内心独白、密码、密钥、登录令牌一概不进工作包。大文件只存一份，靠引用传递。

## 六、去 AI 味策略（审查员承载，绝对关键）

读者和平台对 AI 味零容忍。本团队去 AI 味作为**审查员的能力模块**（Step 6 审查 + Step 7 去 AI 化），不设独立角色，对标得到写作专家团「去 AI 味融入周审稿」的做法。三重保障从源头到终审贯穿：

### 保障 1：源头建立作者站位（撰写前，撰稿人执行）
撰稿人动笔前必须先做四步（来自 humanize-chinese-writing）：
1. 确定文本任务（文体/读者/场景）
2. 建立作者站位（脱离聊天上下文独立成立）
3. 选择组织原则（主线）
4. 写出并检查首稿

### 保障 2：硬规则层（撰写时 + 审查时）
**卡兹克禁用清单**（命中即返工，详见 `style-library/kenny-writer.md`）：
- 禁用词：赋能、抓手、闭环、底座、生态、矩阵、链路、范式、势能、护城河、心智、赛道
- 禁用结构：首先…其次…最后、在当今 AI 快速发展的…、综上所述/总的来说
- 禁用标点：滥用破折号——、滥用冒号：、滥用 emoji、滥用「」（统一用""）
- AI 伪口语扫描：北方方言腔、网络梗堆砌、写作元语言外泄（主菜/收束/锚点/升番）

### 保障 3：四层金字塔审查 + 去 AI 化 + 活人感终审（审查员承载）
- **Step 6 四层金字塔审查**：L1 硬规则扫描（零容忍）/ L2 风格一致性 / L3 内容质量深审 / **L4 活人感终审**（用一个判定问题代替一堆规则——「读完这篇文章，是一个有见识的朋友在跟我聊一件打动他的事，还是 AI 在给我输出信息？」任何一段 AI 味重即返工）
- **Step 7 去 AI 化处理**：三重保障（作者站位复核 + 五层检查 + 卡兹克禁用清单扫描），逐段标注 `ai_taste_score (1-5)`，任何一段 < 3 即返工

### 保障 4：风格学习机制（审查员触发，风格官沉淀）
每次用户修改终稿后回复「继续」：
1. 审查员逐段对比 AI 原稿与用户修改版，提炼 P0/P1/P2 规律
2. 移交风格官评估写入
3. 风格官：P0（同类修改 ≥3 次）→ 经用户确认 → 写入风格库（写入前快照可回滚）

## 六·补、QA 体系（三层 Loop，贯穿全流程）

去 AI 味是「挑错型 QA」（第一层 Loop），已完善。但按吴恩达 Loop Engineering 三层逻辑，第一层是基础但非核心竞争力——真正的壁垒在第二层（QA → PM 转型，定义读者价值）和第三层（长期上下文资产沉淀，构建别人无法短期复刻的上下文优势）。

完整 QA 框架见 [workflow/qa-framework.md](workflow/qa-framework.md)（v1.1，含 13 个方法论吸收），核心要点：

| 层级 | 目标 | 触发节点 | 承载角色 | 方法论吸收 |
|---|---|---|---|---|
| 第一层 · 执行层 | 文章「没有错」（事实准、引用全、无 AI 味） | Gate A1 / B / C | 审查员（已有） | 吸收 1-4：偏见消除 / FActScore / CRITIC / Evals 可演进 |
| 第二层 · 读者价值层 | 文章「值得读」（读者花 10 分钟得到什么） | Gate 0 / A / E | 秘书 + 审查员 L3 升级 | 吸收 5-8：MT-Bench 多轮 / HHH 三维 / 多 Agent 辩论 / 个性化评判 |
| 第三层 · 长期竞争力层 | 「强化作者辨识度 + 积累复利资产」 | Gate E 后 + 季度复盘 | 风格官 + 作者本人 | 吸收 9-13：Reflexion / Self-Rewarding / 风格指纹 / ExpertPrompting / 认知投降检测 |

**上下文优势 = 品味的工程化替换**（吴恩达核心洞察，v1.1 术语对齐）：吴恩达原话用「上下文优势」替换「品味」概念，因为品味暗示不可传授，上下文优势指向可操作路径。写作团队中已有的上下文资产（作者 Gate 把关决策 / 终稿修改理由 / 风格官 Voice Calibration / 选题官认知剪刀差 / HKRT 价值承诺 / 踩坑反思日志 / Agent 自评记录 / 风格指纹漂移度 / 认知投降指标）需显式归档（11 项），让上下文优势可积累、可回看、可迭代、可被 AI 复用。

**角色融合**（不新增角色）：审查员 QA → PM（L3 从挑错升级为读者价值验证）；秘书调度 → PM（Gate 0 做读者价值定义 + 认知投降指标监控）；风格官画像维护 → 上下文资产管理（Gate E 后做长期竞争力评估 + 指纹漂移检测）。

**停止条件机制**（v1.1 新增）：第三层 QA 引入「认知投降检测」——不只评估上下文资产好不好，还评估上下文资产机制是否还在运转。一旦作者认知投降（Gate 通过时间持续缩短 / 修改 diff 持续减少 / 选题拍板介入率持续为 0），强制深度介入做校准。这是去 AI 味战略的长期保障。

## 八、平台适配（排版师出口）

统一用 Markdown 作为 Agent 间与跨平台的中介格式，出口端分别渲染：

| 平台 | 适配要点 | 工具 |
|---|---|---|
| 微信公众号 | H2+H5 双层标题、段落 2-4 行、加粗关键短语、引用块金句、灰色页脚、中英空格 | wechat 规范 + doocs/md |
| 知乎 | 偏分析向、结论先行-证据-场景三层结构、保留完整图文、代码块优化 | mdnice |
| 小红书 | 精简版、封面大字标题、6-9 图轮播、文字 ≤500 字、emoji 分隔、话题标签 | 专用格式化分支 |

**交付要求**：终稿按平台生成 `-wechat / -zhihu / -xhs` 后缀版本，**排版自检清单逐项输出 ✅/❌ 后交付**，确保用户可直接复制粘贴发布。公众号走草稿箱 API + 人工确认；知乎/小红书无开放 API，输出可复制内容由人工发布（不自动发布，规避合规风险）。

## 九、五种文章类型

| 类型 | 定位 | 读者看完应能说 |
|---|---|---|
| case | 产品体验型 / 落地证据 | 「原来是这样落地的」 |
| insight | 现象解读型 / 认知型 | 「本质原来是这样」 |
| lab | 调查实验型 / 探索型 | 「太有意思了，我也想试」 |
| review | 工具分享型 / 决策型 | 「我的场景应该用 X，因为…」 |
| tutorial | 方法论分享型 / 教学型 | 「照着走了一遍，真的做出来了」 |

## 十二、启动方式

对秘书说一件事即可，例如：
- 「最近 AI 圈有啥可写的」（→ 触发热点雷达）
- 「写一篇关于 XX 的 insight，Kenny 风格，公众号+知乎」（→ 带选题直进）
- 「把这份素材写成 case」（→ 素材契约激活）

秘书会先复述意图、追问 3 层，确认后才正式启动。**不会一句话就开跑。**
