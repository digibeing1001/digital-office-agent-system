# 多平台排版规范

> 排版师执行依据。统一 Markdown 中间表示，出口端分别渲染。
> 目标：用户拿到即可复制粘贴发布，无需二次调整。

---

## 微信公众号（wechat 规范，来源 1001-wechat v3.0）

### 核心原则
**只调格式不改内容。** 发现内容问题记录反馈，不擅自修改。

### 1. 标题与信息栏
- 页面 title = 主标题，不在正文重复
- 正文首行：`<span color="gray">作者 · YYYY.MM.DD</span>`

### 2. 标题层级（双层骨架）
- **H2（`##`）**：主章节标题
- **H5（`#####`）**：章节内次级/过渡小标题
- 长文优先「H2 + H5」双层结构，避免单层标题撑全篇

### 3. 段落节奏
- 每段 **2-4 行**，超 100 字必须按语义拆分
- 段间留空行
- 禁止 4 行以上密集长段落

### 4. 重点突出
- **加粗**用于关键短语（不是整句），每段最多 1-2 处
- 核心金句单独成段用引用块 `>`
- 引用块前后留空行，独立呈现

### 5. 图文搭配
- 封面图位于信息栏之后
- 正文内插图：图下独立一段灰色说明 `<span color="gray">图片说明</span>`

### 6. 链接处理
- 链接保留原文可点击格式
- 重要外链可单独成段或放入引用块

### 7. 分隔与导航
- H2 章节之间可加 `---`
- 章节内部一般不用分隔线

### 8. 收尾结构
- 古诗词独立成段（保留作者签名，按风格库要求）
- `---` 分隔线
- 灰色页脚三行：
  - `<span color="gray">✍️ 作者：{name}</span>`
  - `<span color="gray">📮 本文仅代表个人观点，决策请独立思考。</span>`
  - `<span color="gray">🔔 关注「{号名}」，一起探索 AI 时代的无限可能。</span>`

### 9. 格式细节
- 中英文之间加空格
- 统一中文双引号""
- 数字/价格/百分比保持英文原样（如 `$49`、`50%`）

### 工具
- **doocs/md**（https://github.com/doocs/md）：Markdown→微信图文渲染，在线版 https://md.doocs.tech
- **mdnice 墨滴**（https://mdnice.com）：多主题样式、一键排版、免费图床

## 知乎

### 适配要点
- 偏分析向，展开产品思考逻辑
- 保留完整图文
- 代码块优化，长文用目录
- 结论先行-证据-场景三层结构（知乎高赞公式）

### 排版规范
- 支持原生 HTML
- 代码块语法高亮优化
- 长文加目录（TOC）
- 段落首句结论先行（≤35 字，含具体动作或判断）
- 每自然段 ≤2 行，单段只承载 1 个信息单元
- 转折词（但是/其实/这里要注意）前方必须换段
- 专业术语首次出现括号加注 ≤8 字

### 工具
- **mdnice**：支持知乎一键排版

## 小红书

### 适配要点
- 精简版，封面大字标题
- 6-9 图轮播
- 文字 ≤500 字
- emoji 分隔段落
- 加话题标签

### 标题公式
- 含钩子词 + 数字 + 痛点
- 实例：「新手必学…」「3 步搞定…」「亲测有效…」

### 结构（BFD 法则 + 钩子-痛点-方案-证据）
1. **钩子**（3 秒暴击痛点）
2. **痛点**（Behavior：描述行为）
3. **方案**（神转折）
4. **证据**（Feeling：描述感受；Decision：给决策）
5. 可复制行动指令

### 排版规范
- 图文比例 3:7
- 段落分明，用 emoji 分隔
- 重点加粗/特殊符号
- 标题须含关键词以进广场收录
- 热门标签 #

## 交付命名

终稿按平台生成后缀版本：
- `YYYYMMDD-标题-wechat`（公众号）
- `YYYYMMDD-标题-zhihu`（知乎）
- `YYYYMMDD-标题-xhs`（小红书）

属性：状态=待发布、平台、风格、观点、读者。

## 发布方式（保留人工确认）

| 平台 | 方式 |
|---|---|
| 公众号 | 草稿箱 API（`/cgi-bin/draft/add`）+ 手机端确认群发 |
| 知乎 | 无开放 API，输出可复制内容由人工发布 |
| 小红书 | 无开放 API，输出可复制内容由人工发布 |

**不自动发布到任何平台**，最终发布由人工确认，规避合规风险。
